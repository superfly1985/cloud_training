## v2.2.10 (2026-04-22)

### 摘要
- 本次版本重点完成“单一环境检查方法”落地：环境检查、修复复检、训练前门禁统一复用同一规则源与同一检查实现，避免多版本逻辑分叉。
- 训练产物交付流程升级为“统一命名 + 自动完整打包”：每次训练自动生成 `命名_时间戳.zip`，并在 ZIP 内统一整理模型与关键配置文件。
- 针对 TFLite 导出偶发失败新增容错与重试机制，保证主流程稳定；即使 TFLite 失败也不阻断训练成功判定与 ZIP 产物生成。

### 详细变更
- **环境检查逻辑统一**
  - 统一使用 `_get_env_spec` 作为规则源，`_run_env_check_with_spec` 作为唯一检查入口
  - `check_environment`、`fix_environment`、`start_training` 训练门禁复用同一检查实现
  - `get_python_cmd_with_fallback` 调整为“只探测不安装”，避免隐式改环境
- **修复流程收敛**
  - `fix_environment` 改为“先检查后修复”，仅安装缺失项，不再全量重复安装
  - 修复后强制复检并回写环境指纹，确保检查状态与实际环境一致
- **训练阶段禁止自动修复**
  - 移除训练阶段的 `torch/numpy/opencv` 强制重装逻辑
  - 训练阶段仅输出诊断信息，不再执行环境安装动作
- **导出与成功判定优化**
  - 训练后自动导出 ONNX/TFLite，并支持 TFLite 多轮重试（含退避等待）
  - 成功判定调整为：`ONNX` 必需、`TFLite` 可选（失败仅告警）
- **完整 ZIP 自动打包**
  - 自动生成完整产物包，命名为 `命名_时间戳.zip`
  - ZIP 内文件统一追加同一命名标签，避免多批次文件混淆
  - 修复已含标签文件的“重复叠加命名”问题
- **命名规则增强**
  - `dataset.yaml` 在 ZIP 内重命名为 `标记物名称_命名_时间戳.yaml`，降低与 `args_*.yaml` 混淆
  - 命名时间戳统一改为东八区（UTC+8）
- **下载能力扩展**
  - 模型查询/下载列表增加 `.zip` 识别与展示
  - 下载 `.zip` 时保留远端原始命名，不做二次改名
- **版本升级**
  - v2.2.9 → v2.2.10

## v2.2.9 (2026-04-22)

### 新增功能
- **导出依赖并入环境检查**: 将 ONNX/TFLite 导出相关依赖纳入云端环境检查与修复流程
  - 新增检查项：`onnx`、`onnxsim`、`tensorflow`、`flatbuffers`、`protobuf`、`h5py`
  - 缺失时可通过修复流程自动安装并复检
- **训练后自动导出**: 训练脚本在训练完成后自动执行模型导出
  - 自动导出 `ONNX` 与 `TFLite`
  - 导出结果写入结构化日志，便于 GUI 判定与排障
- **模型下载扩展格式支持**: 下载模型功能支持 `*.pt`、`*.onnx`、`*.tflite`
  - 文件查询范围扩展到三类模型文件
  - 保存对话框支持按扩展名自动匹配

### 变更
- **训练完成判定升级**: 将“训练成功”升级为“训练 + 导出均成功”
  - 仅当检测到 `FINAL_STATUS: SUCCESS` 时显示完成状态与完成通知
  - 任一导出失败会明确标记为失败，避免“看似成功但产物不全”
- **版本升级**: v2.2.8 → v2.2.9
