云端清理工具使用说明
========================

概述
----
此工具用于清理云端服务器上的训练模型、数据集和临时文件，避免影响后续训练任务。

文件说明
--------
1. cloud_cleanup.py        - 主清理脚本
2. cloud_config.json       - 配置文件
3. run_cloud_cleanup.bat   - Windows批处理启动脚本
4. 云端清理工具使用说明.txt - 本说明文件

使用步骤
--------

第一步：配置云服务器连接信息
1. 编辑 cloud_config.json 文件
2. 修改以下配置项：
   - hostname: 你的云服务器IP地址
   - username: 用户名（通常是 root）
   - password: 密码（如果使用密码认证）
   - key_filename: 密钥文件路径（如果使用密钥认证）

配置示例：
{
  "cloud_server": {
    "hostname": "192.168.1.100",
    "port": 22,
    "username": "root",
    "password": "your_password",
    "key_filename": null,
    "timeout": 30
  }
}

第二步：确认清理目录
检查 cloud_config.json 中的 directories_to_clean 列表，确认要清理的目录：
- /root/yolo_dataset/      - YOLO数据集目录
- /root/runs/train/        - 训练结果目录
- /root/runs/detect/       - 检测结果目录
- /root/runs/val/          - 验证结果目录
- /root/ultralytics_cache/ - Ultralytics缓存目录

第三步：运行清理工具
方式1：双击运行 run_cloud_cleanup.bat
方式2：在命令行中运行 python cloud_cleanup.py

第四步：确认清理操作
1. 工具会先扫描云端文件并显示预览
2. 显示将要删除的目录和文件信息
3. 需要输入 'YES' 确认执行清理操作

安全特性
--------
1. 配置验证：启动前验证连接配置
2. 预览功能：显示将要删除的文件和目录
3. 确认机制：需要明确确认才会执行删除
4. 保护目录：系统重要目录受到保护
5. 详细日志：记录所有操作和错误信息
6. 清理报告：生成详细的清理报告

输出文件
--------
1. cloud_cleanup_YYYYMMDD_HHMMSS.log - 操作日志
2. cloud_cleanup_report_YYYYMMDD_HHMMSS.json - 清理报告

注意事项
--------
1. 此操作将永久删除云端文件，请谨慎操作
2. 建议在清理前备份重要的训练模型
3. 确保云服务器网络连接稳定
4. 如果使用密钥认证，确保密钥文件路径正确
5. 首次使用建议先在测试环境验证

故障排除
--------
1. 连接失败：检查IP地址、端口、用户名和密码
2. 权限错误：确保用户有足够权限删除文件
3. 网络超时：检查网络连接，可能需要增加timeout值
4. 配置错误：检查 cloud_config.json 文件格式是否正确

联系支持
--------
如遇到问题，请检查生成的日志文件获取详细错误信息。
