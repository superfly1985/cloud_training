## 2. 环境管理策略

### 2.1 为什么必须双环境分离？

**旧项目踩坑经验**：训练环境（PyTorch + CUDA）与转换环境（TensorFlow）在同一环境下安装会导致：

| 冲突类型 | 具体表现 |
|---------|---------|
| protobuf 版本漂移 | TensorFlow 安装时会升级 protobuf，导致 PyTorch 运行异常 |
| CUDA 版本冲突 | PyTorch 绑定 CUDA 12.4，TensorFlow 绑定不同 CUDA 版本 |
| numpy 版本冲突 | TensorFlow 对 numpy 版本有严格限制，可能降级导致 PyTorch 不兼容 |
| 运行时环境漂移 | 训练后安装转换依赖，导致已验证的训练环境被污染 |

### 2.2 双环境方案（已验证）

基于旧项目实际验证的双环境方案：

```
miniforge3 (base)                              # 服务器预装 Python 3.10
│
├── cloud-training (训练+服务环境)              # 主环境
│   │
│   ├── Web 服务                               # FastAPI + Uvicorn + WebSocket
│   ├── GPU 训练                               # torch 2.6.0+cu124 + ultralytics
│   ├── ONNX 导出                              # onnx 1.16.1 + onnxsim
│   └── 训练后处理                              # 重命名、打包、产物管理
│
└── cloud-conversion (转换专用环境)             # 仅用于 TFLite 转换
    │
    ├── TFLite 转换链路                         # ultralytics + onnx2tf + tensorflow
    ├── 格式支持                                # ai-edge-litert + tf-keras + h5py
    └── 序列化依赖                              # protobuf 5.x + flatbuffers
```

### 2.4 转换流程（跨环境）

```
训练完成 (cloud-training 环境)
    │
    ▼
best.pt → best.onnx
  (ultralytics.export(format='onnx'), 在训练环境执行)
    │
    ▼
调用转换环境执行 TFLite 导出
  (subprocess 调用 cloud-conversion 的 Python)
    │
    ▼
best.pt → best_fp32.tflite + best_fp16.tflite
  (ultralytics.export(format='tflite'), 在转换环境执行)
  内部链路: YOLO → onnx2tf → SavedModel → TFLite
    │
    ▼
产物归档到 runs/.../package/
清理中间文件 (*_saved_model/, 临时 .tflite)
```

### 2.5 环境调用方式

转换时通过 subprocess 调用转换环境的 Python（复刻旧项目方案）：

```python
import subprocess

CONVERT_PYTHON = "/root/miniforge3/envs/cloud-conversion/bin/python"

def convert_to_tflite(model_path, imgsz=640, data_yaml=""):
    script = (
        "from ultralytics import YOLO\n"
        "import os, glob, json\n"
        f"model = YOLO('{model_path}')\n"
        f"model.export(format='tflite', imgsz={imgsz}, batch=1, int8=False, half=True, nms=True, data='{data_yaml}')\n"
        f"model.export(format='tflite', imgsz={imgsz}, batch=1, int8=False, half=False, nms=True, data='{data_yaml}')\n"
    )
    result = subprocess.run(
        [CONVERT_PYTHON, "-c", script],
        capture_output=True, text=True, timeout=3600
    )
    return result
```

### 2.6 转换环境门禁

复刻旧项目的转换前检查逻辑：

```python
def check_conversion_env(python_cmd):
    """检查转换环境是否可用"""
    check_script = (
        "import json\n"
        "res = {'ok': False, 'error': ''}\n"
        "try:\n"
        "    from ultralytics import YOLO\n"
        "    import onnx2tf\n"
        "    import tensorflow\n"
        "    res['ok'] = True\n"
        "except Exception as e:\n"
        "    res['error'] = str(e)\n"
        "print(json.dumps(res))\n"
    )
    result = subprocess.run(
        [python_cmd, "-c", check_script],
        capture_output=True, text=True, timeout=30
    )
    return json.loads(result.stdout.strip())
```

## 3. 安装步骤

### 3.1 确认 miniforge3 可用

```bash
/root/miniforge3/bin/python --version
# 期望输出: Python 3.10.x

# 如果未安装
wget https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh
bash Miniforge3-Linux-x86_64.sh -b -p /root/miniforge3
```

### 3.2 创建项目目录

```bash
mkdir -p /opt/cloud-training
cd /opt/cloud-training
mkdir -p data/datasets data/runs data/pretrained data/temp/uploads data/temp/conversion data/temp/split data/logs data/db
```

### 3.3 创建训练环境

```bash
/root/miniforge3/bin/conda create -n cloud-training python=3.10 -y
source /root/miniforge3/bin/activate cloud-training

# Phase 1: 系统库
apt-get update -qq && apt-get install -y libgl1-mesa-glx libglib2.0-0 libusb-1.0-0 -qq

# Phase 2: 核心训练包
pip install pyyaml numpy==1.26.4 opencv-python==4.7.0.72 pillow
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
pip install ultralytics matplotlib

# Phase 3: ONNX 导出链
pip install onnx==1.16.1 onnxsim onnxruntime-gpu

# Phase 4: Web 服务
pip install fastapi uvicorn websockets
```

### 3.4 创建转换环境

```bash
/root/miniforge3/bin/conda create -n cloud-conversion python=3.10 -y
source /root/miniforge3/bin/activate cloud-conversion

# Phase 1: 基础依赖
pip install pyyaml numpy==1.26.4 opencv-python==4.7.0.72 pillow

# Phase 2: 转换核心（含完整 TensorFlow，这是 onnx2tf 的必需依赖）
pip install ultralytics
pip install "tensorflow>=2.19,<2.20" "tf-keras>=2.19,<2.20"
pip install "onnx2tf>=1.28.8,<1.29" ai-edge-litert sng4onnx onnx_graphsurgeon
pip install onnx onnxsim
pip install "protobuf>=5,<6" h5py flatbuffers

# 验证安装
python -c "from ultralytics import YOLO; import onnx2tf; import tensorflow; print('转换环境OK')"
```

> **为什么转换环境安装完整 TensorFlow 而非 tensorflow-cpu？**
> - 旧项目验证：`onnx2tf` 依赖完整 TensorFlow，CPU 版本可能缺少部分符号
> - 转换环境独立于训练环境，不会污染 PyTorch CUDA
> - 转换时设置 `CUDA_VISIBLE_DEVICES=-1` 强制 CPU 执行，避免占用 GPU

### 3.5 清理 base 环境与缓存（重要）

双环境创建完成后，**必须清理 base 环境中的冗余包和 pip 缓存**，否则磁盘空间不足：

```bash
# 1. 清理 base 环境中的训练/转换包（这些已在专用环境中安装）
source /root/miniforge3/bin/activate base
pip uninstall -y torch torchvision torchaudio ultralytics tensorflow tf-keras \
  onnx2tf ai-edge-litert onnx onnxsim onnxruntime onnxruntime-gpu \
  sng4onnx onnx_graphsurgeon flatbuffers h5py protobuf \
  json2onnx onnx2json onnxoptimizer sam4onnx sbi4onnx scc4onnx \
  scs4onnx sde4onnx sed4onnx simple_onnx_processing_tools \
  sio4onnx sit4onnx sna4onnx snc4onnx snd4onnx sne4onnx \
  sog4onnx sor4onnx spo4onnx ssc4onnx ssi4onnx svs4onnx \
  2>/dev/null || true

# 2. 清理 conda 包缓存
/root/miniforge3/bin/conda clean --all -y

# 3. 清理所有环境的 pip 缓存（回收约 8-10 GB）
/root/miniforge3/bin/pip cache purge 2>/dev/null || true
/root/miniforge3/envs/cloud-training/bin/pip cache purge 2>/dev/null || true
/root/miniforge3/envs/cloud-conversion/bin/pip cache purge 2>/dev/null || true

# 4. 验证磁盘空间
df -h / | tail -1
# 期望可用 > 8 GB
```

> **为什么必须清理 base 环境？**
> - 旧项目踩坑：base 环境中 torch + tensorflow 混合安装，导致 protobuf 版本漂移
> - 双环境方案的核心就是隔离，base 环境不应保留任何训练/转换专用包
> - 清理后可回收约 8-10 GB 磁盘空间（pip 缓存 + base 环境冗余包）
> - base 环境仅保留 conda 自身的基础包，作为环境管理器使用

## 4. 启动服务

### 4.1 直接启动（开发模式）

```bash
cd /opt/cloud-training
source /root/miniforge3/bin/activate cloud-training
python run.py
```

### 4.2 后台启动（生产模式）

```bash
cd /opt/cloud-training
source /root/miniforge3/bin/activate cloud-training
nohup python run.py > data/logs/app.log 2>&1 &
```

### 4.3 Systemd 服务配置

创建 `/etc/systemd/system/cloud-training.service`:

```ini
[Unit]
Description=YOLO Cloud Training WebUI
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/cloud-training
Environment=PATH=/root/miniforge3/envs/cloud-training/bin
ExecStart=/root/miniforge3/envs/cloud-training/bin/python run.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable cloud-training
sudo systemctl start cloud-training
sudo systemctl status cloud-training
```

## 8. 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 服务无法启动 | 端口被占用 | 修改配置端口或关闭占用端口的程序 |
| Python 版本不对 | 系统 Python 3.8 | 确保使用 miniforge3 的 cloud-training 环境 |
| 无法访问 GPU | CUDA 驱动未安装 | 安装 NVIDIA 驱动和 CUDA 工具包 |
| 训练失败 | 显存不足 | 减小 batch_size 或 imgsz |
| 上传失败 | 磁盘空间不足 | 清理临时文件或扩展磁盘 |
| ONNX 导出失败 | onnx 未安装 | 在训练环境: `pip install onnx==1.16.1 onnxsim` |
| TFLite 转换失败 | 转换环境不可用 | 检查 cloud-conversion 环境: `conda activate cloud-conversion && python -c "from ultralytics import YOLO; import onnx2tf"` |
| protobuf 冲突 | 训练环境被污染 | 确认训练环境未安装 tensorflow，必要时重建环境 |
| conda 命令找不到 | miniforge3 未加入 PATH | 使用完整路径 `/root/miniforge3/bin/conda` |
