# 前端设计规范

基于 Web Interface Guidelines 合规要求汇总，适用于 YOLO 云端训练平台 WebUI。

---

## 1. 布局系统

### 1.1 整体架构

采用 Grafana 仪表盘风格：左侧固定侧边栏 + 顶部状态摘要栏 + 主内容区。

| 区域 | 宽度/高度 | 说明 |
|------|----------|------|
| 侧边栏 | 240px（可折叠至 64px） | 导航菜单，折叠时仅显示图标 |
| 顶部状态栏 | 48px | 系统状态灯、GPU/磁盘/运行中任务摘要 |
| 主内容区 | 剩余空间 | 各功能模块内容 |

### 1.2 响应式断点

| 断点 | 宽度 | 行为 |
|------|------|------|
| 大屏 | >= 1200px | 侧边栏展开 + 主内容区 |
| 中屏 | 768px - 1199px | 侧边栏折叠为图标 + 主内容区 |
| 小屏 | < 768px | 侧边栏隐藏 + 汉堡菜单触发 |

### 1.3 内容区最大宽度

主内容区内容最大宽度 1200px，居中显示，避免超宽屏幕下阅读困难。

## 2. 排版规范

### 2.1 字体

| 用途 | 字体 | 回退 |
|------|------|------|
| 界面文字 | system-ui | -apple-system, "Segoe UI", Roboto, sans-serif |
| 代码/日志 | "JetBrains Mono" | "Fira Code", "Cascadia Code", monospace |
| 数值数据 | 继承界面字体 | `font-variant-numeric: tabular-nums` |

### 2.2 字号层级

| 层级 | 大小 | 字重 | 用途 |
|------|------|------|------|
| H1 | 24px | 600 | 页面标题 |
| H2 | 20px | 600 | 区域标题 |
| H3 | 16px | 600 | 卡片标题 |
| Body | 14px | 400 | 正文 |
| Caption | 12px | 400 | 辅助说明、时间戳 |

### 2.3 行高与间距

- 正文行高：1.5
- 标题行高：1.3
- 段落间距：16px
- 列表项间距：8px

| Success | #16a34a | 通过、完成、正常状态 |
| Warning | #d97706 | 警告、非阻断问题 |
| Danger | #dc2626 | 错误、阻断、删除 |
| Info | #0891b2 | 提示、帮助信息 |

### 3.2 中性色阶

| 层级 | 色值 | 用途 |
|------|------|------|
| Gray-50 | #f9fafb | 页面背景 |
| Gray-100 | #f3f4f6 | 卡片背景 |

- 交互元素：hover 态加深 10%，active 态加深 20%

## 4. 交互组件规范

### 4.1 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | 实心填充 | 主操作（开始训练、确认） |
| Secondary | 描边 | 次要操作（取消、重置） |
| Danger | 红色实心 | 破坏性操作（删除） |
| Ghost | 无边框无背景 | 辅助操作（展开/折叠） |

按钮文案规范：
- 使用动词开头："下载产物包" 而非 "下载"
- 请求中显示 spinner + 文案追加 "..."："开始训练..."
- 禁用态添加 `aria-disabled="true"` + `title` 说明原因

### 4.2 表单

| 规则 | 要求 |
|------|------|
| 标签 | 每个输入必须有 `<label>`，使用 `for` + `id` 关联 |
| 占位符 | 统一以 "..." 结尾："请选择数据集..." |
| 必填标记 | 使用 `*` 标记，并在表单顶部说明 "* 为必填项" |
| 验证 | 客户端即时验证 + 提交时全量验证 |
| 错误提示 | 内联显示在字段下方，提交时聚焦第一个错误字段 |
| 数值输入 | `<input type="number">` + `autocomplete="off"` + `spellcheck="false"` |
| 下拉选择 | `<select>` + 第一个 option 为占位提示 |

### 4.3 弹窗

| 规则 | 要求 |
|------|------|
| 打开 | 焦点移入弹窗，Tab 循环不跳出 |
| 关闭 | Esc 键 + 点击遮罩 + 关闭按钮 |
| 确认弹窗 | 标题说明操作 + 正文说明后果 + 按钮文案具体 |
| 滚动穿透 | 弹窗内 `overscroll-behavior: contain` |

### 4.4 表格

| 规则 | 要求 |
|------|------|
| 语义 | `<table>` + `<thead>` + `<tbody>` + `<th scope>` |
| 排序 | 可排序列添加 `aria-sort` 属性 |
| 行选择 | `<input type="checkbox">` + `<label>` |
| 空状态 | 表格区域显示空状态插图 + 说明文字 + 引导按钮 |
| 长文本 | `max-width` + `text-overflow: ellipsis` + hover 显示全文 |

### 4.5 分页组件

分页组件用于数据集表格、训练任务列表、产物包列表的客户端分页。

**布局结构**：

```
共 N 条  [首页] [上一页] 1 2 3 ... 10 [下一页] [末页]  跳转至第 [__] 页 [跳转]
```

**各页面分页规格**：

| 页面 | 每页条数 | 分页数据源 |
|------|---------|-----------|
| 数据集表格 | 8 | `pagedDatasets` |
| 训练任务列表 | 15 | `pagedTasks` |
| 产物包列表 | 12 | `pagedPackages` |

**交互规则**：

| 规则 | 说明 |
|------|------|
| 搜索重置 | 搜索或筛选变更时自动重置到第 1 页 |
| 页码越界 | 删除数据导致总页数减少时自动修正当前页 |
| 页码省略 | 超过 10 页时显示 `…`，使用全局 `calcVisiblePages()` 函数 |
| 跳转输入 | `<input type="number">` + `autocomplete="off"` + `aria-label` |
| 键盘支持 | 跳转输入框支持 Enter 键触发跳转 |
| 当前页高亮 | 当前页码按钮使用 Primary 色填充 |

**无障碍要求**：

| 元素 | 要求 |
|------|------|
| 首页/末页按钮 | `aria-label="首页"` / `aria-label="末页"` |
| 上一页/下一页按钮 | `aria-label="上一页"` / `aria-label="下一页"` |
| 页码按钮 | `aria-label="第 N 页"`，当前页添加 `aria-current="page"` |
| 跳转输入框 | `aria-label="跳转页码"` |
| 禁用按钮 | `aria-disabled="true"` + `title` 说明原因 |

### 4.6 Spinner 加载图标

Spinner 用于表示正在进行的异步操作，包括按钮提交中和运行中状态徽章。

**CSS 定义**：

```css
.spinner {
  display: inline-block;
  width: 14px;
  height: 14px;
  border: 2px solid rgba(255, 255, 255, 0.3);
  border-top-color: #fff;
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
  vertical-align: middle;
}
```

**使用场景**：

| 场景 | 尺寸 | 边框宽度 | 附加样式 |
|------|------|---------|---------|
| 按钮内 | 14px | 2px | 无 |
| 状态徽章内 | 10px | 1.5px | `margin-right: 4px` |

**无障碍**：

- `prefers-reduced-motion` 下 spinner 停止旋转（`animation-duration: 0.01ms`）
- 状态信息不依赖颜色，同时使用文字标签 + spinner 图标

## 5. 无障碍合规

### 5.1 必须满足

| 条目 | 要求 |
|------|------|
| 语义 HTML | 使用 `<nav>`、`<main>`、`<article>`、`<section>` 等语义标签 |
| ARIA 标签 | 交互元素必须有 `aria-label` 或关联 `<label>` |
| 焦点管理 | 所有交互元素可通过 Tab 键访问，焦点样式清晰可见 |
| 键盘操作 | 支持 Enter/Space 激活、Esc 关闭、方向键导航 |
| 状态通知 | 动态内容使用 `aria-live` 区域通知屏幕阅读器 |
| 颜色独立 | 信息传达不依赖颜色，必须同时使用文字或图标 |

### 5.2 进度条规范

```html
<div role="progressbar"
     aria-valuenow="45"
     aria-valuemin="0"
     aria-valuemax="100"
     aria-label="训练进度">
  <div style="width: 45%"></div>
</div>
```

### 5.3 实时更新区域

```html
<div aria-live="polite" aria-label="训练日志">
  <!-- 日志内容 -->
</div>
```

## 7. 数值显示规范

### 7.1 数字与单位

数字与单位之间使用不间断空格（`\u00A0` 或 `&nbsp;`）分隔：

| 场景 | 正确 | 错误 |
|------|------|------|
| 文件大小 | 14.5 MB | 14.5MB |
| 百分比 | 92.34% | 92.34 % |
| 时间 | 1h 45m | 1h45m |
| GPU 显存 | 24 GB | 24GB |

### 7.2 等宽数字

所有数值数据使用 `font-variant-numeric: tabular-nums`，确保表格和列表中数字对齐：

```css
.numeric {
  font-variant-numeric: tabular-nums;
}
```

### 7.3 精度规范

| 数据类型 | 精度 | 示例 |
|---------|------|------|
| mAP | 2 位小数 | 92.34% |
| Loss | 3 位小数 | 0.234 |
| 文件大小 | 1 位小数 | 14.5 MB |
| 训练耗时 | 整数 | 1h 45m |

## 9. 焦点样式

### 9.1 默认焦点

```css
:focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
}
```

### 9.2 焦点规则

- 不移除默认焦点样式（禁止 `outline: none`）
- 使用 `:focus-visible` 仅在键盘导航时显示焦点
- 焦点颜色使用 Primary 色，确保与背景对比度 >= 3:1
- 焦点偏移 2px，避免与元素边框重叠

### 10.1 布局

| 缺失 | `.status-error` | `#dc2626`（红色） | 环境缺失 |

.status-indicator {

### 10.4 无障碍

## 11. Bbox Canvas 标注可视化

### 11.1 Canvas 叠加结构

```
┌─────────────────────────────┐
│  .preview-container         │
│  ┌───────────────────────┐  │
│  │  <img> 图片           │  │
│  │  <canvas> Bbox 叠加层  │  │  ← 绝对定位，与图片完全对齐
│  └───────────────────────┘  │
│  ■ 类别名 ← 颜色图例       │
└─────────────────────────────┘
```

- Canvas 使用 `position: absolute` 叠加在 `<img>` 上方
- Canvas 的 `left`/`top`/`width`/`height` 与图片显示区域精确对齐
- 图片使用 `object-fit: contain` 居中显示，Canvas 同步计算偏移

### 11.2 Bbox 绘制样式

| 元素 | 样式 |
|------|------|
| 边框 | 2px 实线，颜色按类别分配 |
| 类别标签背景 | 类别颜色填充，圆角矩形 |
| 类别标签文字 | 白色 12px sans-serif，加粗 |
| 标签位置 | 默认在框上方，溢出顶部时移至框内 |

**线宽自适应**：

```javascript
var lineWidth = Math.max(2, displayW / 300);
```

**字号自适应**：

```javascript
var fontSize = Math.max(10, Math.min(16, displayW / 40));
```

### 11.3 颜色调色板

预定义 12 色调色板，按 `class_id % 12` 分配：

| 索引 | 色值 | 颜色 |
|------|------|------|
| 0 | #ef4444 | 红 |
| 1 | #22c55e | 绿 |
| 2 | #3b82f6 | 蓝 |
| 3 | #eab308 | 黄 |
| 4 | #d946ef | 品红 |
| 5 | #06b6d4 | 青 |
| 6 | #f97316 | 橙 |
| 7 | #ec4899 | 粉 |
| 8 | #8b5cf6 | 紫 |
| 9 | #14b8a6 | 青绿 |
| 10 | #6366f1 | 靛 |
| 11 | #a16207 | 棕 |

### 11.4 DPR 适配

Canvas 需适配高 DPI 屏幕：

```javascript
var dpr = window.devicePixelRatio || 1;
canvas.width = displayW * dpr;
canvas.height = displayH * dpr;
canvas.style.width = displayW + "px";
canvas.style.height = displayH + "px";
ctx.setTransform(dpr, 0, 0, dpr, 0, 0);
```

### 11.5 交互行为

| 操作 | 行为 |
|------|------|
| 点击图片 | 加载图片 + 请求标注，绘制 Bbox |
| 切换图片 | 清除旧 Canvas，绘制新标注 |
| 窗口缩放 | 监听 `resize` 事件，自动重绘 |
| 无标注图片 | 隐藏 Canvas，仅显示原图 |

## 12. 上传进度面板

### 12.1 面板布局

```
┌─────────────────────────────────────────┐
│  上传数据集                              │
│                                         │
│  文件: dataset_v2.zip (500 MB)          │
│                                         │
│  ████████████████░░░░░░░░░░  67%        │
│                                         │
│  速度: 12.5 MB/s                        │
│  已传: 335 MB / 500 MB                  │
│  剩余: 约 13 秒                         │
│                                         │
│              [取消上传]                  │
└─────────────────────────────────────────┘
```

### 12.2 进度条样式

| 属性 | 值 |
|------|-----|
| 高度 | 8px |
| 圆角 | 4px |
| 背景色 | `#e5e7eb`（Gray-200） |
| 填充色 | `#2563eb`（Primary） |
| 过渡动画 | `transition: width 300ms ease-in-out` |

### 12.3 速度显示格式

| 速度范围 | 格式 | 示例 |
|---------|------|------|
| < 1 KB/s | `X B/s` | `512 B/s` |
| 1 KB/s - 1 MB/s | `XX.X KB/s` | `256.3 KB/s` |
| > 1 MB/s | `XX.X MB/s` | `12.5 MB/s` |

### 12.4 剩余时间格式

| 时间范围 | 格式 | 示例 |
|---------|------|------|
| < 60 秒 | `XX 秒` | `45 秒` |
| 1-60 分 | `X 分 XX 秒` | `2 分 13 秒` |
| > 60 分 | `X 时 X 分` | `1 时 23 分` |

### 12.5 速度计算

使用滑动窗口（8 个样本）平滑瞬时速度，避免数字跳动：

```javascript
self._speedSamples.push(instantSpeed);
if (self._speedSamples.length > 8) self._speedSamples.shift();
var sum = 0;
for (var i = 0; i < self._speedSamples.length; i++) sum += self._speedSamples[i];
self.uploadSpeed = sum / self._speedSamples.length;
```

### 12.6 无障碍

- 进度条添加 `role="progressbar"` + `aria-valuenow` + `aria-valuemin="0"` + `aria-valuemax="100"`
- 速度和剩余时间使用 `aria-live="polite"` 区域
- 取消按钮在请求中添加 `aria-disabled="true"`
