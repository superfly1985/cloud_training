# 系统初始化模块

---

## 1. 设计目标

旧版每次训练前都要重复检查环境（Python、CUDA、依赖包），因为是通过 SSH 远程执行，无法持久化状态。新版部署在服务器本地，**开机即自动检查并修复环境**，首次访问前端时强制进入初始化页面，**未完成初始化则阻断所有功能操作**。

## 3. 检查项清单

| 检查项 | 检查内容 | 自动修复 | 阻断级别 |
|--------|---------|---------|---------|
| **Python 环境** | Python 3.10+ 是否存在（优先检测 miniforge3/conda，其次系统 Python）| 否（必须手动安装 miniforge3）| 阻断 |
| **CUDA 驱动** | nvidia-smi 是否可用 | 否 | 阻断 |
| **GPU 可用性** | CUDA 设备是否可访问 | 否 | 阻断 |
| **训练环境** | cloud-training conda env 是否存在且依赖完整 | 是（自动创建+安装）| 阻断 |
| **转换环境** | cloud-conversion conda env 是否存在且依赖完整 | 是（自动创建+安装）| 阻断 |
| **PyTorch** | torch 是否已安装且 CUDA 可用（训练环境）| 是（pip install）| 阻断 |
| **Ultralytics** | 是否已安装（训练环境）| 是（pip install）| 阻断 |
| **ONNX** | 是否已安装（训练环境）| 是（pip install）| 阻断 |
| **TensorFlow** | 是否已安装（转换环境）| 是（pip install）| 阻断 |
| **onnx2tf** | 是否已安装（转换环境）| 是（pip install）| 阻断 |
| **转换门禁** | ultralytics + YOLO + onnx2tf + tensorflow 均可导入（转换环境）| 否 | 阻断 |
| **环境隔离** | 训练环境中不存在 tensorflow/onnx2tf | 是（清理 base 冗余包）| 阻断 |
| **缓存清理** | pip 缓存 + conda 缓存已清理 | 是（自动清理）| 非阻断 |
| **磁盘空间** | 剩余空间是否 > 8 GB | 否（仅警告）| 非阻断 |
| **数据目录** | /opt/cloud-training/data 是否存在 | 是（自动创建）| 非阻断 |

## 5. 核心 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/init/status` | GET | 获取初始化状态 |
| `/api/v1/init/check` | POST | 手动触发检查 |
| `/api/v1/init/fix` | POST | 自动修复问题 |
| `/api/v1/init/progress` | GET | 获取修复进度 |

### 5.1 状态响应

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | 状态：unknown/checking/ready/partial/failed/repairing |
| `checks` | CheckItem[] | 检查项列表 |
| `last_check` | datetime | 最后检查时间 |

**CheckItem 结构**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 检查项名称 |
| `status` | string | 状态：pass/fail/warning |
| `message` | string | 状态说明 |
| `auto_fixable` | bool | 是否可自动修复 |

---

## 6. 前端路由守卫

1. 应用启动时调用 `/api/v1/init/status` 获取初始化状态
2. 如果状态为 `failed` 或 `partial`，强制跳转到 `/init` 页面
3. `/init` 页面显示检查项列表和修复按钮
4. 修复完成后自动跳转到主页

## 7. 初始化页面界面设计

### 7.0 全屏沉浸式布局

初始化页面采用全屏沉浸式设计，不显示侧边栏和顶部状态栏，聚焦于环境检查流程。

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│              YOLO 云端训练平台                               │
│              系统初始化                                      │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  检查项列表 / 修复进度 / 结果展示                            │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  操作按钮区域                                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.0.1 动画规范

| 阶段 | 动画 | 实现 |
|------|------|------|
| 页面进入 | 淡入 + 上移 20px | `@keyframes fadeInUp` 300ms ease-out |
| 检查项状态变更 | 行背景色闪烁 | `transition: background-color 200ms` |
| 进度条推进 | 平滑增长 | `transition: width 500ms ease-in-out` |
| 修复完成跳转 | 淡出当前页 + 淡入主页 | View Transition API `startViewTransition()` |
| 状态图标切换 | 缩放弹跳 | `transform: scale(1.2)` 150ms 后回弹 |

### 7.0.2 无障碍要求

| 元素 | 要求 |
|------|------|
| 进度条 | `role="progressbar"` + `aria-valuenow` + `aria-label="初始化进度"` |
| 检查项列表 | `<ul role="list">` + 每项 `<li>` |
| 状态变更 | `aria-live="polite"` 容器包裹检查项列表 |
| 修复按钮 | 请求中显示 spinner + `aria-disabled="true"` |
| 页面跳转 | 跳转前 `aria-busy="true"`，完成后聚焦主内容区 |

**init_state.json 结构**:

```json
{
  "status": "ready",
  "last_check": "2026-05-12T10:30:00Z",
  "checks": [
    {
      "name": "Python环境",

      "message": "Python 3.10.12 已安装",
      "auto_fixable": false
    },
    {
      "name": "训练环境",
      "status": "pass",
      "message": "cloud-training 已就绪 (torch 2.6.0+cu124 | ultralytics | onnx)",

    },
    {
      "name": "转换环境",
      "status": "pass",
      "message": "cloud-conversion 已就绪 (tensorflow 2.19 | onnx2tf | ai-edge-litert)",

    }
  ]
}
```
