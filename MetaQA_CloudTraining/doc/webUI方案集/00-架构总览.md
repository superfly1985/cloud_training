## 1. 为什么选 Web 版

| 痛点（旧版 PyQt+SSH） | Web 版解决方案 |
|----------------------|---------------|
| 数据集上传走 SFTP，慢且易中断 | HTTP 分片上传 ZIP，支持断点续传 |
| 训练日志通过 SSH 轮询读取 | 本地文件直接读取 + WebSocket 实时推送 |
| TFLite 转换需远程 SSH 执行命令 | 本地 subprocess 调用，路径简单直接 |
| PyInstaller 打包分发麻烦 | 一次部署，浏览器访问 |
| 仅 Windows 客户端 | 任何设备、任何系统 |
| 多用户无法共享 | 天然多用户支持 |

## 2. 技术栈选型（简化设计）

**设计原则**：这是个工具软件，越简单越容易维护越好。

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| **后端框架** | FastAPI + Uvicorn | 异步高性能、自动 OpenAPI 文档、Python 生态 |
| **前端** | Vue 3 CDN + Bootstrap 5 | **无构建工具、直接HTML引入CDN、简单实用** |
| **实时通信** | WebSocket (原生) | 训练日志实时推送 |
| **数据库** | SQLite | 单用户工具，轻量零配置 |
| **文件存储** | 本地文件系统 | 数据集、模型文件本地管理 |
| **部署方式** | 直接运行 Python | 单文件启动，无需 Docker |

> **为什么不选复杂方案？**
> - Vue 3 + Vite + TypeScript + Pinia + Element Plus — 技术栈过于复杂
> - 工具软件不需要企业级组件库、状态管理、类型检查
> - CDN 引入方式可直接修改 HTML 调试，维护成本最低

## 3. 端口规划（连续端口，方便防火墙配置）

| 服务 | 端口 | 说明 |
|------|------|------|
| FastAPI（后端 API + WebSocket + 静态文件） | `8090` | 主服务端口，所有功能合一 |
| TensorBoard（训练监控，可选） | `8091` | 仅需要时启动 |
| 预留扩展 | `8092-8095` | 未来可能的服务 |

> **防火墙配置只需开放 8090-8095**：
> ```bash
> # Linux 防火墙示例
> firewall-cmd --add-port=8090-8095/tcp --permanent
> firewall-cmd --reload
> ```
> 
> 生产环境访问方式：`http://<服务器IP>:8090`
