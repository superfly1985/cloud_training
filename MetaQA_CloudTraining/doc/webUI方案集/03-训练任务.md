## 2. 智能分组算法（分层采样）

**目的**：确保 train/val 中各类别比例一致，避免某类别只在 train 或只在 val 中出现。

### 2.1 算法流程

```
训练启动时
    │
    ▼
┌─────────────────────────┐
│  解析所有标注文件        │ ← 读取 labels/*.txt
│  统计每张图片的类别列表  │ ← 图片可能包含多个类别
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  按主要类别分组          │ ← 每张图片归属其出现最多的类别
│  构建 class_to_images    │ ← {class_0: [img1, img5...], class_1: [img3, img7...]}
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  对每个类别独立采样      │ ← 对 class_to_images[class_id] 随机打乱
│  按 val_ratio 划分       │ ← 前 N 张 → train，后 M 张 → val
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  合并各类别的分组结果    │ ← train_images = ∑ train_class_i
│                          │ ← val_images = ∑ val_class_i
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  生成分组配置文件        │ ← 写入 split_cache.json
│  创建临时目录结构        │ ← temp/split/{session_id}/images/train, val
│  建立软链接              │ ← 指向原图片（不复制，节省空间）
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  更新 dataset.yaml       │ ← 修改 path、train、val 指向临时目录
│  启动 YOLO 训练          │ ← YOLO 读取软链接目录
└─────────┬───────────────┘
          │
          ▼
     训练进行中
          │
          ▼
┌─────────────────────────┐
│  训练完成后清理          │ ← 删除 temp/split/{session_id}
│  保留 split_cache.json   │ ← 记录分组结果，便于复现
└─────────────────────────┘
```

### 2.2 算法说明

**输入参数**:

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `dataset_path` | string | - | 数据集目录路径 |
| `val_ratio` | float | 0.2 | 验证集比例 |
| `seed` | int | 42 | 随机种子（保证可复现） |

**算法步骤**:

1. **解析标注文件**: 读取 `labels/*.txt`，统计每张图片的类别列表
2. **按主要类别分组**: 每张图片归属其出现最多的类别，构建 `class_to_images` 映射
3. **对每个类别独立采样**: 对每个类别的图片列表随机打乱，按 `val_ratio` 划分
4. **合并各类别的分组结果**: `train_images = Σ train_class_i`，`val_images = Σ val_class_i`
5. **生成分组配置文件**: 写入 `split_cache.json`，创建临时目录结构（使用软链接，不复制文件）
6. **更新 dataset.yaml**: 修改 `path`、`train`、`val` 指向临时目录
7. **训练完成后清理**: 删除 `temp/split/{session_id}`，保留 `split_cache.json`

### 2.3 分组结果缓存 (split_cache.json)

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 训练会话 ID |
| `val_ratio` | float | 验证集比例 |
| `seed` | int | 随机种子 |
| `total_images` | int | 总图片数 |
| `train_count` | int | 训练集图片数 |
| `val_count` | int | 验证集图片数 |
| `train_images` | string[] | 训练集图片列表 |
| `val_images` | string[] | 验证集图片列表 |
| `class_distribution` | object | 各类别在 train/val 中的分布统计 |
| `created_at` | datetime | 创建时间 |

## 3. 训练参数配置

### 3.1 基础参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `dataset_id` | string | - | 数据集 ID |
| `version_id` | string | - | 版本 ID（如 v001） |
| `model_name` | string | - | 模型名称（用于命名 runs 目录） |
| `model_size` | string | "n" | 模型大小：n/s/m/l/x |
| `epochs` | int | 100 | 训练轮数 |
| `imgsz` | int | 640 | 输入图片尺寸 |
| `batch` | int | 16 | 批次大小 |
| `lr0` | float | 0.01 | 初始学习率 |
| `val_ratio` | float | 0.2 | 验证集比例（分层采样） |
| `split_seed` | int | 42 | 分组随机种子 |
| `device` | string | "cuda:0" | 训练设备 |
| `patience` | int | 50 | 早停耐心值 |
| `resume` | bool | false | 是否断点续训 |
| `resume_path` | string | null | 续训的 last.pt 路径 |

### 3.2 数据增强参数（预留接口，默认不启用）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `augment` | bool | false | 数据增强总开关 |
| `mosaic` | float | 0.0 | Mosaic 增强概率（0-1） |
| `mixup` | float | 0.0 | MixUp 增强概率（0-1） |
| `hsv_h` | float | 0.0 | HSV 色调调整幅度 |
| `hsv_s` | float | 0.0 | HSV 饱和度调整幅度 |
| `hsv_v` | float | 0.0 | HSV 明度调整幅度 |
| `degrees` | float | 0.0 | 旋转角度范围（度） |
| `translate` | float | 0.0 | 平移比例（0-1） |
| `scale` | float | 0.0 | 缩放比例（0-1） |
| `fliplr` | float | 0.0 | 水平翻转概率（0-1） |
| `flipud` | float | 0.0 | 垂直翻转概率（0-1） |

> **注意**：增强参数已预留接口，但默认全部禁用。如需启用，将对应参数设置为推荐值即可。

## 4. 训练执行流程

```
用户配置训练参数（含分组比例、种子）
        │
        ▼
┌───────────────┐
│  执行智能分组  │ ← stratified_split()
│  创建软链接目录│ ← temp/split/{session_id}/
│  写入分组缓存  │ ← split_cache.json
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  选择预训练模型│
│  配置超参数    │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  生成训练脚本  │ ← 写入 runs/{exp_id}/train_v3.py
│  创建运行目录  │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  启动子进程    │ ← subprocess.Popen
│  记录 PID      │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  WebSocket    │ ← 实时推送 stdout/stderr
│  日志流       │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│  轮询 results.csv│ ← 解析损失曲线数据
│  推送图表数据  │
└───────┬───────┘
        │
        ▼
   训练完成/停止
        │
        ▼
┌───────────────┐
│  记录训练会话  │ ← 写入 runs/registry.json
│  更新模型列表  │
└───────────────┘
```

## 5. 核心 API

### 5.1 训练任务 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/training/start` | POST | 启动训练 |
| `/api/v1/training/{session_id}/stop` | POST | 停止训练 |
| `/api/v1/training/{session_id}/status` | GET | 查询训练状态 |
| `/api/v1/training/{session_id}/logs` | GET | 实时日志（SSE） |
| `/api/v1/training/{session_id}/ws` | WebSocket | 实时日志（WebSocket） |
| `/api/v1/training/{session_id}/chart` | GET | 获取图表数据 |
| `/api/v1/training/history` | GET | 训练历史 |

### 5.2 启动训练请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---------|------|------|--------|------|
| `dataset_id` | string | 是 | - | 数据集 ID |
| `version_id` | string | 是 | - | 版本 ID |
| `model_name` | string | 是 | - | 模型名称（用于命名 runs 目录） |
| `base_model` | string | 否 | "yolov8n.pt" | 预训练模型 |
| `epochs` | int | 否 | 100 | 训练轮数 |
| `imgsz` | int | 否 | 640 | 输入图片尺寸 |
| `batch` | int | 否 | 16 | 批次大小 |
| `lr0` | float | 否 | 0.01 | 初始学习率 |
| `device` | string | 否 | "0" | GPU 设备号 |
| `patience` | int | 否 | 50 | 早停耐心值 |
| `resume` | bool | 否 | false | 是否断点续训 |

### 5.3 训练状态响应

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 训练会话 ID |
| `status` | string | 状态：pending/running/completed/failed/stopped |
| `current_epoch` | int | 当前轮数 |
| `total_epochs` | int | 总轮数 |
| `current_loss` | float | 当前损失 |
| `best_map` | float | 最佳 mAP |
| `elapsed_time` | int | 已用时间（秒） |
| `estimated_remaining` | int | 预计剩余时间（秒） |

### 5.4 图表数据响应

| 字段 | 类型 | 说明 |
|------|------|------|
| `epochs` | int[] | 轮数列表 |
| `box_loss` | float[] | Box 损失 |
| `cls_loss` | float[] | 分类损失 |
| `dfl_loss` | float[] | DFL 损失 |
| `precision` | float[] | 精确率 |
| `recall` | float[] | 召回率 |
| `map50` | float[] | mAP@50 |
| `map50_95` | float[] | mAP@50-95 |

### 7.1 表单无障碍规范

| 元素 | 要求 |
|------|------|
| 数据集选择 | `<label for="dataset-select">` + `<select id="dataset-select" name="dataset" autocomplete="off">` |
| 数值输入 | `<label>` + `<input type="number" name="epochs" autocomplete="off" spellcheck="false">` |
| 占位符 | 统一使用 `...` 结尾：`请选择数据集...` |
| 提交按钮 | 请求中显示 spinner，文案变为 `开始训练...` |
| 错误提示 | 内联显示在字段下方，提交时聚焦第一个错误字段 |
| 必填标记 | 使用 `*` 标记，并在字段旁说明 `* 为必填项` |
| 单选按钮 | `<input type="radio">` + `<label>` 共享点击区域 |

### 7.2 表单验证规则

| 字段 | 验证规则 | 错误提示 |
|------|---------|---------|
| 数据集 | 必填 | "请选择数据集" |
| 训练轮数 | 1-1000 整数 | "训练轮数须为 1-1000 的整数" |
| 批次大小 | 1-128 整数 | "批次大小须为 1-128 的整数" |
| 学习率 | 0.0001-1.0 | "学习率须为 0.0001-1.0" |
| 验证集比例 | 0.1-0.5 | "验证集比例须为 0.1-0.5" |

### 7.3 交互说明

- 数据集选择为必填项，未选择时"开始训练"按钮置灰
- 模型大小切换时显示对应参数量提示（如 n=3.2M, s=11.2M）
- 断点续训勾选后展开路径选择
- 数据增强区域灰显不可操作，标注"功能开发中"
- 提交前验证所有字段，错误内联显示并聚焦第一个错误
- 提交中按钮显示 spinner，防止重复提交

## 10. 运行中状态 Spinner

### 10.1 样式规范

运行中的训练任务在状态徽章内显示旋转加载图标：

```html
<span class="badge" :class="statusBadgeClass(task.status)">
  <span v-if="task.status === 'running'" class="spinner"></span>
  {{ statusText(task.status) }}
</span>
```

### 10.2 CSS 定义

```css
.spinner {
  display: inline-block;
  width: 14px;
  height: 14px;
  border: 2px solid rgba(255, 255, 255, 0.3);
  border-top-color: #fff;
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
  vertical-align: middle;
}

.badge .spinner {
  width: 10px;
  height: 10px;
  border-width: 1.5px;
  margin-right: 4px;
}
```

### 10.3 无障碍

- `prefers-reduced-motion` 下 spinner 停止旋转（`animation-duration: 0.01ms`）
- 状态信息不依赖颜色，同时使用文字标签 + spinner 图标

## 11. 查看产物跳转

### 11.1 按钮规格

| 属性 | 值 |
|------|-----|
| 图标 | `bi-box-seam` |
| 位置 | 查看日志、损失曲线之后 |
| 显示条件 | 仅 `task.status === 'completed'` |
| `aria-label` | "查看产物" |
| `title` | "查看产物" |

### 11.2 跳转逻辑

1. 点击后通过 `task.dataset_name` + `task.version` 在产物列表中查找对应产物包
2. 调用 `this.$root.navigateTo('package')` 切换到产物页
3. 通过根实例传递 `highlightPackageId`，产物页接收后给对应卡片添加高亮状态
4. 若未找到对应产物包，弹出提示"未找到对应产物包"

### 11.3 高亮效果

- 产物卡片添加 `.highlight` 类
- CSS 动画：`@keyframes highlight-pulse`，边框颜色从 `--color-primary` 渐变到透明，持续 2 秒，循环 2 次后自动移除
- 高亮期间卡片自动滚动到可视区域（`scrollIntoView`）
- `prefers-reduced-motion` 下禁用脉冲动画，仅保留边框高亮
