# 目录结构设计

---

## 1. 项目目录结构

`MetaQA_CloudTraining/` 是完全重构的新项目，不沿用旧版任何代码。后端 FastAPI + 前端 Vue 3 CDN，一条命令启动。

仓库根目录目前已经做了产品线分区：

- `MetaQA_CloudTraining/`：当前主产品 Web 版
- `legacy/local_desktop/`：保留可运行、可打包能力的旧本地版
- `test/`：保留的探针脚本与少量样本

本文只描述 `MetaQA_CloudTraining/` 这条 Web 主线的内部结构，不展开旧本地版目录。


**架构要点**：

| 对比项 | 旧版（PyQt+SSH） | 新版（Web） |
|--------|-----------------|------------|
| 启动方式 | PyInstaller exe | **python run.py 一条命令** |
| 前端 | PyQt6 原生控件 | **Vue 3 CDN 无构建，Grafana 风格单页应用** |
| 前端文件 | 混在 Python 中 | **独立 static/ 目录，8 个 JS + 1 个 CSS + 1 个 HTML** |
| 后端框架 | 无（SSH 直连） | **FastAPI + Uvicorn** |
| 业务逻辑 | 混在 main.py | **core/ Manager 层，api/ 薄路由层** |
| 实时通信 | SSH 轮询 | **WebSocket 原生推送** |
| 数据库 | 无 | **SQLite 轻量零配置** |
| 部署方式 | Windows exe | **Linux 服务器，浏览器访问** |

## 2. 各目录职责说明

### 2.1 `app/api/` — API 路由层（薄层）

职责：接收请求 → 参数校验 → 调用 core → 返回响应。不包含业务逻辑。

| 文件 | 路由前缀 | 调用的 Manager |
|------|---------|---------------|
| `init.py` | `/api/v1/init` | `InitManager` |
| `datasets.py` | `/api/v1/datasets` | `DatasetManager` |
| `upload.py` | `/api/v1/upload` | `UploadManager` |
| `training.py` | `/api/v1/training` | `TrainingManager` + `MonitorManager` |
| `packages.py` | `/api/v1/packages` | `PackageManager` |
| `system.py` | `/api/v1/system` | `SystemManager` |

### 2.2 `app/core/` — 核心业务逻辑（Manager 层）

职责：封装所有业务逻辑，被 api 层调用。每个 Manager 对应一个业务域。

| Manager | 职责 | 关键方法 |
|---------|------|---------|
| `InitManager` | 环境检查、自动修复 | `check()`, `fix()`, `get_status()` |
| `DatasetManager` | 数据集 CRUD、合并、删除图片 | `list()`, `create()`, `merge()`, `delete_images()` |
| `UploadManager` | 分片上传、断点续传 | `init_upload()`, `upload_chunk()`, `complete_upload()` |
| `TrainingManager` | 训练分组、脚本生成、subprocess | `start()`, `stop()`, `get_history()` |
| `MonitorManager` | 日志推送、曲线数据 | `stream_logs()`, `get_chart_data()` |
| `PackageManager` | 模型转换、打包、下载 | `list()`, `convert()`, `download()`, `delete()` |
| `SystemManager` | GPU、磁盘、环境信息 | `get_gpu()`, `get_disk()`, `get_info()` |

### 2.3 `static/` — 前端静态文件

FastAPI 通过 `StaticFiles` 中间件托管，开发时修改即生效，无需构建。

| 文件 | 职责 |
|------|------|
| `index.html` | 单页应用入口，引入 Vue 3 / Bootstrap Icons CDN |
| `css/style.css` | 全局样式、CSS 变量、组件样式、Bbox Canvas、上传进度面板 |
| `js/app.js` | Vue 根实例、侧边栏导航、全局函数、环境状态轮询 |
| `js/api.js` | API 请求封装（含 getImageLabels 等新接口） |
| `js/dashboard.js` | 概览页（GPU/磁盘/训练统计/环境信息） |
| `js/dataset-tab.js` | 数据集管理（表格分页/搜索/多选删除/Bbox 绘制预览） |
| `js/training-tab.js` | 训练任务（表格分页/状态筛选/产物跳转） |
| `js/package-tab.js` | 产物包管理（卡片分页/搜索/高亮跳转） |
| `js/system-tab.js` | 系统状态详情（环境检查/自动修复） |
| `js/modals.js` | 弹窗组件（新建训练/实时日志/损失曲线/产物详情/上传进度） |
| `js/mock-api.js` | 开发用 Mock 数据，生产环境不加载 |

### 2.4 `data/` — 数据持久化

运行时由应用自动创建，不纳入版本控制。

| 目录 | 内容 | 清理策略 |
|------|------|---------|
| `datasets/` | 数据集（images + labels + meta.json） | 用户手动删除 |
| `runs/` | 训练结果（weights + results.csv + package/） | 用户手动删除 |
| `pretrained/` | 预训练模型缓存（yolov8n.pt 等） | 自动管理 |
| `temp/` | 临时文件（上传/转换/分组） | 任务完成后自动清理 |
| `logs/` | 应用日志 | 按大小轮转，保留 10 份 |
| `db/` | SQLite 数据库 | 持久保留 |

### 2.5 `deploy_tool/` — 一键部署工具

独立于主应用的部署工具，用于在本地 Windows 电脑上运行，通过 SSH 将项目代码部署到远程 Linux 服务器。

| 文件 | 职责 |
|------|------|
| `deploy_manager.py` | 部署核心逻辑：SSH 连接、zip 全量上传覆盖、systemd 自启动配置、服务管理 |
| `deploy_gui.py` | 部署工具 GUI（Tkinter）：服务器信息输入、连接测试、12 步状态指示器、实时日志、进度显示 |

**部署工具特性**：

| 特性 | 说明 |
|------|------|
| 一键部署 | 填入 IP/端口/用户名/密码，测试连接后一键部署 |
| 文件上传 | 本地打 zip 后单包上传，远端解压覆盖代码目录 |
| 步骤指示器 | 12 步部署流程实时显示状态（运行中/成功/失败） |
| 自启动配置 | 自动写入 `systemd` unit 并启用服务自启动 |
| 部署中断恢复 | 中断后重新部署，自动跳过已完成步骤 |

## 3. 云端数据目录结构

### 3.1 设计原则

**一个项目一个训练集，支持增量操作**：
- 不做多版本，节省空间
- 可删除单张图片 + 标注
- 可上传新 ZIP 合并到现有数据集（增量上传）
- 微量修改无需重新上传全部

### 3.2 目录结构

**数据集平铺存储，训练时智能分组**：
- images/ 和 labels/ 平铺存放，不预先分组
- 训练启动时按配置动态分组
- 分组采用分层采样，确保各类别均匀分布

## 4. 数据集元数据 (meta.json)

### 4.1 字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `name` | string | 数据集名称 | "焊点漏包" |
| `description` | string | 数据集描述 | "焊点漏包检测数据集" |
| `image_count` | int | 图片总数 | 680 |
| `label_count` | int | 标注总数 | 680 |
| `classes` | string[] | 类别列表 | ["焊点漏包"] |
| `num_classes` | int | 类别数量 | 1 |
| `fingerprint` | string | 数据集指纹(MD5) | "a1b2c3d4e5f6" |
| `created_at` | datetime | 创建时间(ISO 8601) | "2026-05-11T10:30:00Z" |
| `last_modified` | datetime | 最后修改时间 | "2026-05-12T14:00:00Z" |
| `history` | array | 操作历史记录 | 见下表 |

### 4.2 history 数组元素结构

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `action` | string | 操作类型 | create / merge / delete |
| `count` | int | 操作数量(创建时) | 初始图片数 |
| `added` | int | 新增数量(合并时) | 新增图片数 |
| `removed` | int | 删除数量(删除时) | 删除图片数 |
| `time` | datetime | 操作时间 | ISO 8601 格式 |

### 4.3 示例结构

```json
{
  "name": "焊点漏包",
  "image_count": 680,
  "classes": ["焊点漏包"],
  "history": [
    {
      "action": "create",
      "count": 450,
      "time": "2026-05-11T10:30:00Z"
    },
    {
      "action": "merge",
      "added": 230,
      "time": "2026-05-11T14:00:00Z"
    }
  ]
}
```

## 5. 与旧版目录的关系

| 旧版路径 | 新版路径 | 说明 |
|---------|---------|------|
| `main.py`（PyQt6 编排） | `app/main.py`（FastAPI 入口） | 完全重写，框架不同 |
| `src/core/*_manager.py` | `app/core/*_manager.py` | 完全重写，SSH→本地调用 |
| `src/ui/*.py`（PyQt6 窗口） | `static/js/*.js`（Vue 3 组件） | 完全重写，桌面→Web |
| `cloud_training_config.json` | `config/app_config.yaml` | 格式改为 YAML |
| `cloud_training.spec` | 不再需要 | Web 版无需 PyInstaller 打包 |
| `cloud-training-webui/` | `static/` | 前端移入项目内，FastAPI 托管 |
