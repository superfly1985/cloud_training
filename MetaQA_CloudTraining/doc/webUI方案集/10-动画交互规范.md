# 动画交互规范

基于 View Transitions API + CSS 动画，定义 YOLO 云端训练平台 WebUI 的动画与交互规范。

---

## 1. 设计原则

### 1.1 动画目的

- **反馈**：确认用户操作已被接收
- **引导**：引导用户注意力到变化区域
- **连贯**：保持界面状态切换的视觉连续性

### 1.2 动画禁忌

- 不使用纯装饰性动画
- 不使用闪烁、旋转等分散注意力的效果
- 不使用 `animation-iteration-count: infinite` 的持续性动画（进度 spinner 除外）
- 尊重 `prefers-reduced-motion` 系统设置

---

## 2. View Transitions API

### 2.1 页面导航切换

侧边栏导航切换时使用 View Transitions API 实现平滑过渡：

```javascript
async function navigateTo(page) {
  if (!document.startViewTransition) {
    switchPage(page);
    return;
  }

  const transition = document.startViewTransition(() => {
    switchPage(page);
  });

  await transition.finished;
}
```

### 2.2 CSS 视图过渡样式

```css
::view-transition-old(root) {
  animation: fadeOut 150ms ease-out;
}

::view-transition-new(root) {
  animation: fadeIn 150ms ease-in;
}

@keyframes fadeOut {
  from { opacity: 1; }
  to { opacity: 0; }
}

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
```

### 2.3 列表项过渡

数据集列表、训练任务列表、产物包列表的增删动画：

```css
.dataset-card {
  view-transition-name: var(--card-id);
}

::view-transition-old(.dataset-card) {
  animation: slideOutLeft 200ms ease-in;
}

::view-transition-new(.dataset-card) {
  animation: slideInRight 200ms ease-out;
}
```

### 2.4 侧边栏折叠/展开

```css
.sidebar {
  view-transition-name: sidebar;
}

::view-transition-group(sidebar) {
  animation-duration: 200ms;
}

::view-transition-old(sidebar) {
  width: 240px;
}

::view-transition-new(sidebar) {
  width: 64px;
}
```

## 3. CSS 过渡规范

### 3.1 时长标准

| 类别 | 时长 | 用途 |
|------|------|------|
| 即时反馈 | 100ms | 按钮 hover、焦点切换 |
| 短过渡 | 150ms | 颜色变化、透明度变化 |
| 标准过渡 | 200ms | 展开/折叠、位移 |
| 长过渡 | 300ms | 页面切换、弹窗出现 |
| 强调过渡 | 500ms | 进度条推进、复杂形变 |

### 3.2 缓动函数

| 名称 | 函数 | 用途 |
|------|------|------|
| 默认 | `ease` | 通用过渡 |
| 进入 | `ease-out` | 元素出现 |
| 退出 | `ease-in` | 元素消失 |
| 弹性 | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 图标状态切换 |

### 3.3 通用过渡声明

```css
.interactive {
  transition: background-color 150ms ease,
              color 150ms ease,
              border-color 150ms ease,
              box-shadow 150ms ease;
}

.transformative {
  transition: transform 200ms ease-out,
              opacity 200ms ease-out;
}
```

## 4. 组件动画

### 4.1 弹窗

| 阶段 | 动画 | 时长 |
|------|------|------|
| 遮罩出现 | `opacity: 0 -> 1` | 150ms ease |
| 内容进入 | `opacity: 0 -> 1` + `translateY(16px) -> 0` | 200ms ease-out |
| 内容退出 | `opacity: 1 -> 0` + `translateY(0) -> -8px)` | 150ms ease-in |
| 遮罩消失 | `opacity: 1 -> 0` | 150ms ease |

```css
.modal-overlay {
  transition: opacity 150ms ease;
}

.modal-content {
  transition: opacity 200ms ease-out,
              transform 200ms ease-out;
}

.modal-content.entering {
  opacity: 0;
  transform: translateY(16px);
}

.modal-content.visible {
  opacity: 1;
  transform: translateY(0);
}
```

### 4.2 侧边栏导航

| 阶段 | 动画 | 时长 |
|------|------|------|
| 选中态切换 | 左侧指示条滑动 | 200ms ease-out |
| 文字折叠 | `width` + `opacity` | 200ms ease |
| 图标居中 | `transform: translateX()` | 200ms ease-out |

### 4.3 表格行

| 场景 | 动画 | 时长 |
|------|------|------|
| 行 hover | 背景色变化 | 100ms ease |
| 行选中 | 左侧边框出现 | 150ms ease-out |
| 行删除 | 高度收缩 + 透明度 | 200ms ease-in |
| 行新增 | 高度展开 + 透明度 | 200ms ease-out |

### 4.4 进度条

| 场景 | 动画 | 时长 |
|------|------|------|
| 进度推进 | `width` 变化 | 500ms ease-in-out |
| 状态变更 | 背景色变化 | 200ms ease |
| 完成闪烁 | `opacity` 闪烁 1 次 | 300ms |

### 4.5 卡片

| 场景 | 动画 | 时长 |
|------|------|------|
| hover | `box-shadow` 增强 + `translateY(-2px)` | 150ms ease |
| 加载中 | 骨架屏脉冲 | 1.5s ease-in-out infinite |
| 出现 | `opacity` + `translateY(8px)` | 200ms ease-out |

## 5. 初始化页面动画

### 5.1 页面进入

```css
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.init-page {
  animation: fadeInUp 300ms ease-out;
}
```

### 5.2 检查项状态变更

```css
.check-item {
  transition: background-color 200ms ease;
}

.check-item.pass {
  background-color: rgba(22, 163, 74, 0.08);
}

.check-item.fail {
  background-color: rgba(220, 38, 38, 0.08);
}
```

### 5.3 状态图标切换

```css
@keyframes iconPop {
  0% { transform: scale(1); }
  50% { transform: scale(1.2); }
  100% { transform: scale(1); }
}

.check-icon.updated {
  animation: iconPop 150ms ease;
}
```

### 5.4 初始化完成跳转

```javascript
async function transitionToMain() {
  if (!document.startViewTransition) {
    window.location.hash = '#/dashboard';
    return;
  }

  const transition = document.startViewTransition(() => {
    document.getElementById('init-page').style.display = 'none';
    document.getElementById('main-app').style.display = 'flex';
  });

  await transition.finished;
  window.location.hash = '#/dashboard';
}
```

## 7. 无障碍动画

### 7.1 尊重减少动画偏好

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### 7.2 动画替代方案

| 原始动画 | 减少动画模式 |
|---------|------------|
| 淡入淡出 | 直接显示/隐藏 |
| 滑动 | 直接切换 |
| 缩放弹跳 | 直接切换状态 |
| 进度条动画 | 直接更新数值 |

### 7.3 安全动画属性

优先使用 `transform` 和 `opacity` 进行动画，避免触发重排：

| 安全（仅合成层） | 不安全（触发重排） |
|-----------------|------------------|
| `transform` | `width` / `height` |
| `opacity` | `top` / `left` |
| `filter` | `margin` / `padding` |

例外：进度条 `width` 变化不可避免，使用 `will-change: width` 优化。

## 8. 性能约束

### 8.1 动画帧率

- 所有动画必须保持 60fps
- 单个动画属性变更不超过 16ms 计算时间
- 复杂动画使用 `will-change` 提升合成层

### 8.2 并发动画限制

- 同屏同时进行的动画不超过 3 个
- 列表项动画使用交错延迟（stagger），间隔 50ms
- 大量元素变更时（如列表刷新），使用批量过渡而非逐项动画

### 8.3 内存管理

- 动画结束后移除 `will-change` 属性
- View Transition 完成后清理 `view-transition-name`
- 避免在 `requestAnimationFrame` 中创建新对象
