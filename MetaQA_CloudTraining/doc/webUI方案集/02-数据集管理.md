## 1. 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **ZIP 分片上传** | 选择 ZIP → 自动分片上传，支持断点续传 | **P0** |
| **上传进度与速度** | 实时显示上传百分比、速度、已传/总量、剩余时间 | **P0** |
| **上传前预检查** | 检测重复文件，跳过重复节省带宽 | **P0** |
| 创建数据集 | 上传初始 ZIP → 解压 → 验证 → 创建 | P0 |
| **合并新增** | 上传新 ZIP → 合并到现有数据集 | **P0** |
| **删除图片** | 选中图片 → 删除图片 + 对应标注 | **P0** |
| **Bbox 绘制预览** | 图片预览区 Canvas 叠加绘制标注框 + 类别名 + 颜色图例 | **P0** |
| 查看统计 | 图片数、类别分布、标注质量 | P1 |
| 数据集导出 | 打包下载完整数据集 | P2 |

**上传方式说明**：
- **只支持 ZIP 包上传**（不支持文件夹直接上传）
- 原因：ZIP 打包简单、传输稳定、支持分片和断点续传
- 用户操作：本地文件夹 → 手动压缩为 ZIP → Web UI 上传

## 2. 上传前预检查（节省带宽）

**目的**：上传前检测重复文件，避免浪费带宽上传已存在的文件。

**原理**：
- 前端读取 ZIP 文件名列表（不解压，只读取索引）
- 发送到服务器对比现有数据集
- 发现重复时弹窗提示，可选择跳过重复文件

### 2.1 预检查流程

```
用户选择 ZIP 文件 → 点击上传
        │
        ▼
┌─────────────────────────────────────┐
│  前端用 JSZip 读取文件名列表          │
│  （不解压，只读取 ZIP 索引）           │
│  提取 images 目录下的文件名           │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  POST /api/v1/upload/precheck        │
│  发送文件名列表到服务器               │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  服务器对比现有数据集文件名            │
│  返回重复列表 + 新文件列表             │
└─────────────┬───────────────────────┘
              │
              ▼
        ┌─────┴─────┐
        │           │
   无重复        有重复
        │           │
        ▼           ▼
┌──────────┐ ┌─────────────────────┐
│ 直接进入  │ │ 弹窗提示：           │
│ 分片上传  │ │ 发现 N 个重复文件    │
│ 流程      │ │ 选择处理策略         │
└──────────┘ └─────────────────────┘
                    │
            ┌───────┴───────┐
            │               │
         skip            overwrite/rename
            │               │
            ▼               ▼
┌──────────────────────┐ ┌──────────────────┐
│ 【节省带宽】          │ │ 上传全部文件     │
│ 只上传新文件          │ │ 后按策略处理     │
│ 跳过重复文件          │ │                  │
│ 示例：500 MB -> 400 MB   │ │                  │
└──────────────────────┘ └──────────────────┘
```

### 2.2 预检查 API 接口

**接口**: `POST /api/v1/upload/precheck`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `dataset_name` | string | 是 | 数据集名称 | "焊点漏包" |
| `mode` | string | 否 | 上传模式 | "create" 或 "merge"，默认 "create" |
| `filenames` | string[] | 是 | ZIP 内的图片文件名列表 | ["img001.jpg", "img002.jpg"] |
| `total_size` | int | 是 | ZIP 文件总大小（字节） | 524288000 (500 MB) |

**响应数据**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `dataset_name` | string | 数据集名称 |
| `has_duplicates` | bool | 是否存在重复文件 |
| `duplicates` | string[] | 重复文件名列表 |
| `new_files` | string[] | 新文件名列表 |
| `duplicate_count` | int | 重复文件数 |
| `new_count` | int | 新文件数 |
| `skip_recommendation` | string | 跳过建议，如"可跳过 20 个文件，节省约 100 MB" |

### 2.3 带宽节省示例

```
场景：ZIP 500 MB，100 张图片，20 张与现有数据集重复

原方案（上传后检测）：
  上传 500 MB -> 检测发现重复 -> 跳过 -> 浪费约 100 MB 带宽

预检查方案（skip 策略）：
  预检发现 20 张重复 → 提示用户 → 用户选择 skip
  -> 只上传 80 张新图片 -> 上传约 400 MB -> 节省 100 MB
```

## 3. 分片上传（主要上传方式）

**用户操作简单，程序自动分片处理**：
- 用户只需选择一个 ZIP 文件
- 前端 JavaScript 自动拆分成 10 MB 分片
- 支持断点续传、进度显示、暂停/继续

### 3.1 完整流程（含预检查）

```
用户选择 ZIP 文件 → 点击上传
        │
        ▼
┌─────────────────────────────────────┐
│  执行预检查（见 2.1）               │
│  检测重复，决定上传范围                │
└─────────────┬───────────────────────┘
              │
              ▼ (确定上传范围后)
┌─────────────────────────────────────┐
│  前端 JavaScript 自动处理            │
│  ───────────────────────────────    │
│  ZIP (5 GB) -> 拆分成 500 个 10 MB 分片 │
│  显示进度条、速度、剩余时间           │
│  支持暂停/继续                       │
└─────────────────┬───────────────────┘
                  │
                  ▼ (逐个上传分片)
┌─────────────────────────────────────┐
│  后端接收分片                        │
│  ───────────────────────────────    │
│  POST /api/v1/upload/chunk          │
│  存入 temp/uploads/{upload_id}/     │
│  chunk_0.part, chunk_1.part, ...    │
│  返回已完成分片列表                   │
└─────────────────┬───────────────────┘
                  │
    ┌─────────────┴─────────────┐
    │                           │
    ▼                           ▼
┌──────────┐              ┌──────────┐
│ 网络中断  │              │ 上传完成  │
│ 断电等    │              │          │
└────┬─────┘              └────┬─────┘
     │                         │
     ▼                         ▼
┌──────────────────┐    ┌──────────────────┐
│ GET /upload/status│    │ POST /upload/    │
│ 获取已完成分片    │    │ complete         │
│ 恢复后继续上传    │    │ 合并所有分片     │
│ 未完成的分片      │    │ → 解压 → 验证    │
└──────────────────┘    └──────────────────┘
```

### 3.2 分片上传 API 接口

**1. 初始化上传** - `POST /api/v1/upload/init`

| 请求参数 | 类型 | 必填 | 说明 | 默认值 |
|---------|------|------|------|--------|
| `filename` | string | 是 | ZIP 文件名 | - |
| `total_size` | int | 是 | 文件总大小（字节） | - |
| `chunk_size` | int | 否 | 分片大小（字节） | 10485760 (10 MB) |
| `dataset_name` | string | 是 | 数据集名称 | - |

**2. 上传分片** - `POST /api/v1/upload/chunk`

| 请求参数 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `upload_id` | string | 是 | 上传会话 ID |
| `chunk_index` | int | 是 | 当前分片索引 |
| `chunk_data` | bytes | 是 | 分片二进制数据 |

**3. 查询上传状态** - `GET /api/v1/upload/status/{upload_id}`

| 响应字段 | 类型 | 说明 |
|---------|------|------|
| `upload_id` | string | 上传会话 ID |
| `total_chunks` | int | 总分片数 |
| `received_chunks` | int[] | 已接收的分片索引 |
| `missing_chunks` | int[] | 缺失的分片索引 |
| `progress` | float | 上传进度（0-1） |

**4. 完成上传** - `POST /api/v1/upload/complete`

| 请求参数 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `upload_id` | string | 是 | 上传会话 ID |
| `file_hash` | string | 是 | 完整文件 MD5 |

| 响应字段 | 类型 | 说明 |
|---------|------|------|
| `success` | bool | 是否成功 |
| `dataset_id` | string | 数据集 ID |
| `extracted_path` | string | 解压后的目录路径 |

## 6. Bbox 标注可视化

### 6.1 功能概述

在数据集图片预览区，通过 Canvas 叠加绘制 YOLO 格式的标注边界框（Bbox），直观展示标注效果。每种类别使用固定颜色，框上方显示类别名称标签。

### 6.2 技术实现

**渲染流程**：

```
用户点击图片 → 加载图片 + 请求标注数据
        │
        ▼
┌─────────────────────────────────────┐
│  API.getImageLabels(dsId, filename) │
│  返回 { labels, class_names }       │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  构建 classColorMap / classNamesMap │
│  按 class_id 分配固定颜色            │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  drawBBoxes() Canvas 绘制           │
│  ───────────────────────────────    │
│  1. 计算图片在容器中的显示尺寸和偏移  │
│  2. 设置 Canvas DPR 缩放            │
│  3. 遍历 labels 绘制：              │
│     - 矩形边框（类别颜色）           │
│     - 类别标签背景 + 文字            │
│  4. 窗口 resize 时自动重绘           │
└─────────────────────────────────────┘
```

**颜色分配算法**：

- 预定义 12 色调色板（红/绿/蓝/黄/品红/青/橙/粉/紫/青绿/靛/棕）
- `class_id` 为数字时：`BBOX_COLORS[classId % 12]`
- `class_id` 为字符串时：哈希取模 `BBOX_COLORS[abs(hash) % 12]`
- 同一类别在不同图片中颜色一致

**坐标转换**（YOLO 格式 → Canvas 像素）：

```
YOLO 格式: x_center, y_center, width, height (归一化 0-1)
Canvas 像素:
  x1 = (x_center - width/2) × displayW
  y1 = (y_center - height/2) × displayH
  w  = width × displayW
  h  = height × displayH
```

**DPR 适配**：

```
Canvas 物理尺寸: displayW × dpr, displayH × dpr
Canvas CSS 尺寸: displayW, displayH
ctx.setTransform(dpr, 0, 0, dpr, 0, 0)
```

### 6.3 标注数据 API

**获取图片标注** - `GET /api/v1/datasets/{ds_id}/images/{filename}/labels`

响应：

| 字段 | 类型 | 说明 |
|------|------|------|
| `labels` | array | 标注列表 |
| `labels[].class_id` | int | 类别 ID |
| `labels[].x_center` | float | 中心点 X（归一化） |
| `labels[].y_center` | float | 中心点 Y（归一化） |
| `labels[].width` | float | 宽度（归一化） |
| `labels[].height` | float | 高度（归一化） |
| `labels[].class_name` | string | 类别名称 |
| `class_names` | object | 类别 ID→名称映射 |

### 6.4 交互行为

| 操作 | 行为 |
|------|------|
| 点击图片 | 加载图片 + 标注，绘制 Bbox |
| 切换图片 | 清除旧 Canvas，绘制新标注 |
| 窗口缩放 | 自动重新计算尺寸并重绘 |

## 7. 上传进度与速度显示

### 7.1 功能概述

分片上传过程中，实时显示上传进度、传输速度、已传/总量和预计剩余时间，提升大文件上传的用户体验。

### 7.2 显示信息

| 信息项 | 格式 | 示例 |
|--------|------|------|
| 上传百分比 | `XX%` | `67%` |
| 传输速度 | `XX.X KB/s` / `XX.X MB/s` | `12.5 MB/s` |
| 已传/总量 | `XXX MB / XXX MB` | `335 MB / 500 MB` |
| 预计剩余 | `XX 秒` / `X 分 XX 秒` / `X 时 X 分` | `2 分 13 秒` |

### 7.3 速度计算算法

```
每上传一个分片后：
  1. 计算瞬时速度 = bytesDelta / elapsed
  2. 加入滑动窗口（最多保留 8 个样本）
  3. 平均速度 = sum(samples) / len(samples)
  4. 剩余时间 = (fileSize - uploadedBytes) / 平均速度
  5. 格式化剩余时间：< 60s 显示秒, < 3600s 显示分秒, 否则显示时分
```

**滑动窗口平滑**：避免单个分片速度波动导致数字跳动，8 个样本窗口提供稳定的速度显示。
