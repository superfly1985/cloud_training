# 数据集管理模块

---

## 1. 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **ZIP 分片上传** | 选择 ZIP → 自动分片上传，支持断点续传 | **P0** |
| **上传前预检查** | 检测重复文件，跳过重复节省带宽 | **P0** |
| 创建数据集 | 上传初始 ZIP → 解压 → 验证 → 创建 | P0 |
| **合并新增** | 上传新 ZIP → 合并到现有数据集 | **P0** |
| **删除图片** | 选中图片 → 删除图片 + 对应标注 | **P0** |
| 查看统计 | 图片数、类别分布、标注质量 | P1 |
| 数据集导出 | 打包下载完整数据集 | P2 |

**上传方式说明**：
- **只支持 ZIP 包上传**（不支持文件夹直接上传）
- 原因：ZIP 打包简单、传输稳定、支持分片和断点续传
- 用户操作：本地文件夹 → 手动压缩为 ZIP → Web UI 上传

## 2. 上传前预检查（节省带宽）

**目的**：上传前检测重复文件，避免浪费带宽上传已存在的文件。

**原理**：
- 前端读取 ZIP 文件名列表（不解压，只读取索引）
- 发送到服务器对比现有数据集
- 发现重复时弹窗提示，可选择跳过重复文件

### 2.1 预检查流程

```
用户选择 ZIP 文件 → 点击上传
        │
        ▼
┌─────────────────────────────────────┐
│  前端用 JSZip 读取文件名列表          │
│  （不解压，只读取 ZIP 索引）           │
│  提取 images 目录下的文件名           │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  POST /api/v1/upload/precheck        │
│  发送文件名列表到服务器               │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  服务器对比现有数据集文件名            │
│  返回重复列表 + 新文件列表             │
└─────────────┬───────────────────────┘
              │
              ▼
        ┌─────┴─────┐
        │           │
   无重复        有重复
        │           │
        ▼           ▼
┌──────────┐ ┌─────────────────────┐
│ 直接进入  │ │ 弹窗提示：           │
│ 分片上传  │ │ 发现 N 个重复文件    │
│ 流程      │ │ 选择处理策略         │
└──────────┘ └─────────────────────┘
                    │
            ┌───────┴───────┐
            │               │
         skip            overwrite/rename
            │               │
            ▼               ▼
┌──────────────────────┐ ┌──────────────────┐
│ 【节省带宽】          │ │ 上传全部文件     │
│ 只上传新文件          │ │ 后按策略处理     │
│ 跳过重复文件          │ │                  │
│ 示例：500 MB -> 400 MB   │ │                  │
└──────────────────────┘ └──────────────────┘
```

### 2.2 预检查 API 接口

**接口**: `POST /api/v1/upload/precheck`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `dataset_name` | string | 是 | 数据集名称 | "焊点漏包" |
| `mode` | string | 否 | 上传模式 | "create" 或 "merge"，默认 "create" |
| `filenames` | string[] | 是 | ZIP 内的图片文件名列表 | ["img001.jpg", "img002.jpg"] |
| `total_size` | int | 是 | ZIP 文件总大小（字节） | 524288000 (500 MB) |

**响应数据**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `dataset_name` | string | 数据集名称 |
| `has_duplicates` | bool | 是否存在重复文件 |
| `duplicates` | string[] | 重复文件名列表 |
| `new_files` | string[] | 新文件名列表 |
| `duplicate_count` | int | 重复文件数 |
| `new_count` | int | 新文件数 |
| `skip_recommendation` | string | 跳过建议，如"可跳过 20 个文件，节省约 100 MB" |

### 2.3 带宽节省示例

```
场景：ZIP 500 MB，100 张图片，20 张与现有数据集重复

原方案（上传后检测）：
  上传 500 MB -> 检测发现重复 -> 跳过 -> 浪费约 100 MB 带宽

预检查方案（skip 策略）：
  预检发现 20 张重复 → 提示用户 → 用户选择 skip
  -> 只上传 80 张新图片 -> 上传约 400 MB -> 节省 100 MB
```

## 3. 分片上传（主要上传方式）

**用户操作简单，程序自动分片处理**：
- 用户只需选择一个 ZIP 文件
- 前端 JavaScript 自动拆分成 10 MB 分片
- 支持断点续传、进度显示、暂停/继续

### 3.1 完整流程（含预检查）

```
用户选择 ZIP 文件 → 点击上传
        │
        ▼
┌─────────────────────────────────────┐
│  执行预检查（见 2.1）               │
│  检测重复，决定上传范围                │
└─────────────┬───────────────────────┘
              │
              ▼ (确定上传范围后)
┌─────────────────────────────────────┐
│  前端 JavaScript 自动处理            │
│  ───────────────────────────────    │
│  ZIP (5 GB) -> 拆分成 500 个 10 MB 分片 │
│  显示进度条、速度、剩余时间           │
│  支持暂停/继续                       │
└─────────────────┬───────────────────┘
                  │
                  ▼ (逐个上传分片)
┌─────────────────────────────────────┐
│  后端接收分片                        │
│  ───────────────────────────────    │
│  POST /api/v1/upload/chunk          │
│  存入 temp/uploads/{upload_id}/     │
│  chunk_0.part, chunk_1.part, ...    │
│  返回已完成分片列表                   │
└─────────────────┬───────────────────┘
                  │
    ┌─────────────┴─────────────┐
    │                           │
    ▼                           ▼
┌──────────┐              ┌──────────┐
│ 网络中断  │              │ 上传完成  │
│ 断电等    │              │          │
└────┬─────┘              └────┬─────┘
     │                         │
     ▼                         ▼
┌──────────────────┐    ┌──────────────────┐
│ GET /upload/status│    │ POST /upload/    │
│ 获取已完成分片    │    │ complete         │
│ 恢复后继续上传    │    │ 合并所有分片     │
│ 未完成的分片      │    │ → 解压 → 验证    │
└──────────────────┘    └──────────────────┘
```

### 3.2 分片上传 API 接口

**1. 初始化上传** - `POST /api/v1/upload/init`

| 请求参数 | 类型 | 必填 | 说明 | 默认值 |
|---------|------|------|------|--------|
| `filename` | string | 是 | ZIP 文件名 | - |
| `total_size` | int | 是 | 文件总大小（字节） | - |
| `chunk_size` | int | 否 | 分片大小（字节） | 10485760 (10 MB) |
| `dataset_name` | string | 是 | 数据集名称 | - |

**2. 上传分片** - `POST /api/v1/upload/chunk`

| 请求参数 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `upload_id` | string | 是 | 上传会话 ID |
| `chunk_index` | int | 是 | 当前分片索引 |
| `chunk_data` | bytes | 是 | 分片二进制数据 |

**3. 查询上传状态** - `GET /api/v1/upload/status/{upload_id}`

| 响应字段 | 类型 | 说明 |
|---------|------|------|
| `upload_id` | string | 上传会话 ID |
| `total_chunks` | int | 总分片数 |
| `received_chunks` | int[] | 已接收的分片索引 |
| `missing_chunks` | int[] | 缺失的分片索引 |
| `progress` | float | 上传进度（0-1） |

**4. 完成上传** - `POST /api/v1/upload/complete`

| 请求参数 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `upload_id` | string | 是 | 上传会话 ID |
| `file_hash` | string | 是 | 完整文件 MD5 |

| 响应字段 | 类型 | 说明 |
|---------|------|------|
| `success` | bool | 是否成功 |
| `dataset_id` | string | 数据集 ID |
| `extracted_path` | string | 解压后的目录路径 |

### 4.3 空状态设计规范

| 元素 | 说明 |
|------|------|
| 插图 | 简洁的空状态 SVG 插图（文件夹 + 上传图标） |
| 标题 | "暂无数据集" |
| 描述 | "上传 ZIP 包创建你的第一个数据集" |
| 操作按钮 | "创建数据集" -- 直接引导用户执行下一步 |
| 次要链接 | "查看使用文档" -- 可选 |

### 4.4 长文本截断规则

| 场景 | 处理方式 | CSS |
|------|---------|-----|
| 数据集名称 | 超过 20 字符截断，hover 显示全名 | `text-overflow: ellipsis; max-width: 200px;` |
| 图片文件名 | 超过 30 字符截断 | `text-overflow: ellipsis; max-width: 280px;` |
| 类别名称列表 | 超过 3 个类别显示 "+N 更多" | `line-clamp: 1` |

### 4.5 无障碍要求

| 元素 | 要求 |
|------|------|
| 图片列表项 | 使用 `<button>` 包裹，添加 `aria-label="选择图片 img001.jpg"` |
| 复选框 | `<input type="checkbox">` + `<label>` 关联，共享点击区域 |
| 预览区域 | `<img>` 添加 `alt="img001.jpg 预览"` |
| 分页按钮 | 添加 `aria-label="上一页"` / `aria-label="下一页"` |
| 数据集选择器 | `<select>` 添加 `<label for="dataset-select">` |
| 删除确认 | 破坏性操作需二次确认弹窗，包含撤销窗口 |

### 4.6 功能说明

- **列表布局**：左侧显示图片列表，加载更快
- **虚拟滚动**：图片列表超过 50 项时使用虚拟滚动（`content-visibility: auto`）
- **单张预览**：点击哪张图片，右侧显示哪张的预览
- **快速查看**：列表显示文件名、文件大小、标注状态
- **批量操作**：支持多选删除
- **数据集表格分页**：数据集选择表格每页 8 条，含首页/末页/页码/跳转控件
- **图片列表分页**：每页 50 张，含页码按钮和跳转输入框
- **等宽数字**：文件大小、图片数量使用 `font-variant-numeric: tabular-nums`

## 5. 核心 API 汇总

### 5.1 数据集管理 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/upload/precheck` | POST | 上传前预检查 |
| `/api/v1/upload/init` | POST | 初始化上传 |
| `/api/v1/upload/chunk` | POST | 上传分片 |
| `/api/v1/upload/status/{upload_id}` | GET | 查询上传状态 |
| `/api/v1/upload/complete` | POST | 完成上传 |
| `/api/v1/datasets` | GET | 列出数据集 |
| `/api/v1/datasets` | POST | 创建数据集（本地扫描） |
| `/api/v1/datasets/{dataset_id}` | DELETE | 删除数据集 |
| `/api/v1/datasets/{dataset_id}/images` | DELETE | 删除图片 |
| `/api/v1/datasets/{dataset_id}/export` | GET | 导出数据集 |

### 5.2 响应格式

**DatasetInfo 结构**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 数据集 ID |
| `name` | string | 数据集名称 |
| `description` | string | 数据集描述 |
| `image_count` | int | 图片总数 |
| `label_count` | int | 标注总数 |
| `classes` | string[] | 类别列表 |
| `class_distribution` | object | 各类别标注数 {类别名: 数量} |
| `last_modified` | string | 最后修改时间 |
