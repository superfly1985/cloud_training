# Loss metrics display design

本设计统一 `MetaQA_CloudTraining` 中训练损失指标的采集、存储与展示方
式。目标是让训练曲线、训练任务列表、产物包列表三处都以
`box_loss`、`cls_loss`、`dfl_loss` 作为主展示指标，并保持新旧数据兼
容。

## 问题概述

当前系统的训练曲线弹窗展示 `box_loss`、`cls_loss`、`mAP@50`，训练任
务列表与产物包列表也都突出显示 `mAP@50`。这带来两个问题：

- 训练阶段最直观的收敛信息分散，损失指标没有完整呈现。
- 后端当前没有持久化 `dfl_loss`，因此前端无法在列表中稳定展示三项损
  失。

本次改造要把“损失值”作为统一口径，同时保留现有 `mAP` 数据供包元数
据和其他功能使用。

## 目标结果

本改造完成后，系统要满足以下结果：

- 训练曲线弹窗只显示三条损失曲线：
  - `box_loss`
  - `cls_loss`
  - `dfl_loss`
- 训练任务列表将原 `mAP@50` 列替换为“损失值”列。
- 产物包列表将原 `mAP@50` 列替换为“损失值”列。
- 两个列表中的损失值单元格都采用固定三行显示：
  - `box: x.xxx`
  - `cls: x.xxx`
  - `dfl: x.xxx`
- 老任务、老产物包、缺失字段的记录不报错，缺失项显示 `-`。

## 数据链路设计

本节定义后端必须补齐的损失指标链路，避免只做前端表面改动。

### 训练任务数据

训练任务数据源仍来自训练目录内的 `results.csv`。监控模块解析时，必须
继续读取以下列，并新增 `dfl_loss`：

- `train/box_loss`
- `train/cls_loss`
- `train/dfl_loss`

刷新训练指标时，后端必须把最新值写回 `training_tasks` 表，并在接口响
应中返回三项损失值。

### 数据库存储

`training_tasks` 表必须新增 `dfl_loss REAL DEFAULT 0` 字段。初始化逻辑
除了 `CREATE TABLE IF NOT EXISTS` 外，还必须兼容已存在数据库：

- 如果旧表不存在 `dfl_loss` 列，需要通过迁移式补列完成兼容。
- 迁移过程必须幂等，多次启动不会报错。

### 产物包数据

产物包列表当前主要来自 `packages` 表和 ZIP 包中的 `info.json`。本次改
造中，产物生成时必须把三项损失写入 `info.json`：

- `box_loss`
- `cls_loss`
- `dfl_loss`

列表接口返回包信息时，优先从 `info.json` 读取三项损失并透传给前端。
如果旧包没有这些字段，则返回 `null` 或缺省值，由前端显示 `-`。

## 前端展示设计

本节定义三处 UI 的统一展示行为。

### 训练曲线弹窗

训练曲线弹窗继续使用现有弹窗和图表结构，但指标改为三条损失曲线：

- `Box Loss`
- `Cls Loss`
- `DFL Loss`

图表缩放、坐标轴和轮询逻辑保持现有行为，不再绘制 `mAP@50`。

### 训练任务列表

训练任务列表中原 `mAP@50` 列替换为“损失值”列。每个单元格固定显示三
行，顺序固定如下：

```text
box: 0.123
cls: 0.045
dfl: 0.678
```

显示规则如下：

- 对已获取到的数值，统一显示 3 位小数。
- 对缺失值、旧数据、训练尚未开始的项，显示 `-`。
- 不在单元格内额外展示 `mAP`。

### 产物包列表

产物包列表中原 `mAP@50` 列同样替换为“损失值”列，布局与训练任务列表
保持一致。新产物包会从 `info.json` 拿到三项损失，旧产物包则按缺失值
显示 `-`。

## 兼容性要求

本改造必须兼容已有数据库和历史包文件。

- 旧数据库启动后能自动补齐 `dfl_loss` 列。
- 旧训练任务在未刷新到 `dfl_loss` 前，列表仍能正常打开。
- 旧产物包因为历史 `info.json` 不含损失字段，列表展示 `-`，但下载、
  删除和详情行为不受影响。
- 不要求强制重建旧包，也不要求回填历史数据库记录。

## 测试策略

本次改造需要覆盖后端和前端关键回归点。

### 后端测试

后端测试至少覆盖以下行为：

- 解析 `results.csv` 时可以读取 `train/dfl_loss`。
- 刷新训练指标时会把 `dfl_loss` 写回任务数据。
- 创建产物包时会把三项损失写入 `info.json`。
- 读取旧产物包信息时，缺失损失字段不会报错。

### 前端测试与校验

前端至少需要完成以下验证：

- 训练曲线仅渲染三条损失线。
- 训练任务列表显示“损失值”三行单元格。
- 产物包列表显示“损失值”三行单元格。
- 缺失值场景显示 `-`，不出现 `undefined`、`NaN` 或布局错位。

## 范围限制

本次改造只处理损失指标的采集、存储和展示，不包含以下内容：

- 不修改训练算法本身。
- 不移除后端已有的 `map50`、`map50_95` 数据字段。
- 不调整产物包详情弹窗中的其他元信息。
- 不改动部署工具和桌面端旧版本代码。
