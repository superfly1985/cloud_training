# 合并结果区美化设计

本文档定义“合并新增数据”成功后的结果展示收口方案。目标是移除顶部
`alert`，把反馈集中到弹窗内部，并让结果区本身足够醒目、易扫读。

## 背景

当前 `merge-dataset-modal` 已经具备两类反馈能力：

- 合并成功后生成摘要文案
- 合并弹窗内部显示结果区

但现状还有两个问题：

- 顶部 `alert` 会打断操作节奏
- 结果区仍然像普通文本区块，不像“成功结果卡片”

用户已经确认新的方向：

- 取消顶部弹窗
- 美化结果区域

## 目标

本次设计只解决合并成功后的视觉反馈问题。

需要达成以下目标：

- 移除顶部 `alert`
- 保留合并弹窗内的结果反馈
- 把结果区提升为更明确的成功卡片
- 让四项统计更容易快速扫描

## 非目标

本次设计不包含以下内容：

- 不修改后端 merge 返回结构
- 不修改创建数据集弹窗
- 不增加全局 toast、通知中心或历史记录
- 不改变合并完成后的刷新时机

## 方案对比

本次确认采用卡片式结果区，而不是保守微调。

### 方案 A：成功卡片 + 2x2 统计块（最终采用）

在弹窗内部渲染一个完整结果卡片：

- 顶部显示成功标题
- 次级区域显示一行摘要
- 底部用 `2 x 2` 统计块展示四项数字

优点：

- 视觉层级清楚
- 四项数据一眼可读
- 替代 `alert` 后仍具备足够提示强度

缺点：

- 比简单文本区块多一点结构和样式

### 方案 B：保留纵向文字结构，只做配色和间距优化

优点：

- 改动最小

缺点：

- 不足以承担“去掉顶部弹窗”后的主反馈角色

### 方案 C：单行 badge 列表

优点：

- 紧凑

缺点：

- 统计项一多时可读性较差

## 最终设计

本次采用 **方案 A**。

### 信息层级

结果区分成三层：

1. 成功标题
2. 摘要说明
3. 四项统计块

这样可以同时满足“快速确认成功”和“快速读取数字”两类需求。

### 标题区

标题区展示明确成功状态，例如：

```text
合并完成
```

标题左侧可保留成功图标，强化状态识别。

### 摘要区

摘要区展示完整自然语言说明，例如：

```text
图片新增 X，图片覆盖 Y，标签新增 A，标签覆盖 B
```

这部分用于承接原本 `alert` 的信息职责，但改为卡片内展示。

### 统计区

统计区使用 `2 x 2` 布局，分别展示：

- 图片新增
- 图片覆盖
- 标签新增
- 标签覆盖

每个统计块包含：

- 简短标题
- 更突出的数字

统计块要使用一致的尺寸、边框和留白，避免像普通文本列表。

## 交互约束

本次设计维持当前交互边界：

- 合并成功后不自动关闭弹窗
- 用户查看结果后手动关闭
- 数据集列表仍然在关闭弹窗后刷新

这样既保留结果可回看能力，也不需要额外改动刷新链路。

## 样式约束

样式需要尽量复用现有变量和视觉体系。

具体要求如下：

- 继续使用现有成功色系
- 使用与卡片系统一致的圆角和边框
- 不引入新的大型动画
- 不把结果区做得比上传进度区更抢眼

结果区应呈现为“成功状态卡片”，而不是“独立新模块”。

## 模块范围

本次改动只涉及以下文件：

- `static/js/modals.js`
- `static/css/style.css`

其中：

- `modals.js` 负责结构和交互
- `style.css` 负责结果卡片和统计块样式

## 验收标准

实现完成后，必须满足以下标准：

1. 合并成功后不再弹出顶部 `alert`
2. 合并结果只在弹窗内部展示
3. 结果区呈现为明确的成功卡片
4. 四项统计以 `2 x 2` 形式展示
5. 结果仍然支持字段缺失时按 `0` 兜底
6. `merge-dataset-modal` 之外的弹窗行为不变

不把这轮需求扩大为全局消息系统改造。
