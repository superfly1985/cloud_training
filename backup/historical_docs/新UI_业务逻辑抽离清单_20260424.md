# 新UI业务逻辑抽离清单（2026-04-24，已实装）

## 1. 盘点范围
- 范围：`main_v3.py`、`src/ui/main_window.py`、`src/core/*`。
- 目标：将业务逻辑下沉到 `core` 管理器，`main_v3.py` 仅保留事件绑定、线程调度、UI回填。

## 3. 新增/重构模块
- 新增 `src/core/environment_manager.py`：环境检查与修复流程。
- 新增 `src/core/network_monitor_manager.py`：Ping 与丢包率解析。
- 新增 `src/core/model_manager.py`：best.pt 查找、下载、TFLite 转换。
- 新增 `src/core/file_manager_manager.py`：远程文件浏览与删除业务封装。
- 扩展 `src/core/server_manager.py`：连接保障、目录递归上传、远程目录列表、远程删除等能力。
- 重构 `src/core/training_manager.py`：训练参数统一为当前 `ConfigManager` 配置结构。

## 4. 架构现状
- `main_v3.py` 已改为编排层：按钮绑定、异步线程、UI状态更新。
- 业务能力基本都已在 `core` 层提供可复用接口。
- MVP 功能均已接入，不再存在“按钮可见但无实现”的占位入口。

## 5. 后续可继续优化（非本轮阻断）
- 文件管理可补充下载、重命名、批量操作。
- 模型下载可增加模型列表选择，而非默认最新 `best.pt`。
- 环境修复可增加可配置依赖清单与镜像源策略。
