# 路径重构说明（Ubuntu 22.04 Docker 支持）

## 一、重构目标
- 保持 GUI 主入口不变：`cloud_training_gui.py`。
- 按运行环境分层路径，避免“预装 PyTorch”逻辑与 Ubuntu Docker 逻辑相互污染。
- 先做“非破坏式重构”：先新增目录骨架，不立即移动历史文件。

## 二、新增目录

```text
configs/
  preinstalled_pytorch/
  docker_ubuntu2204/

deploy/
  docker_ubuntu2204/

scripts/
  preinstalled_pytorch/
  linux/

docs/开发文档/
  preinstalled_pytorch/
  docker_ubuntu2204/
```

## 三、职责定义
- `configs/preinstalled_pytorch/`：传统预装 PyTorch 服务器模式配置。
- `configs/docker_ubuntu2204/`：Ubuntu 22.04 + Docker 模式配置。
- `deploy/docker_ubuntu2204/`：Docker 部署资产（`Dockerfile`、`docker-compose.yml`、`requirements.lock.txt`、`verify_env.py`）。
- `scripts/preinstalled_pytorch/`：预装 PyTorch 路线辅助脚本（如打包、清理、诊断）。
- `scripts/linux/`：Linux/远端脚本（如容器启动、部署检查）。
- `docs/开发文档/preinstalled_pytorch/`：预装 PyTorch 路径相关文档。
- `docs/开发文档/docker_ubuntu2204/`：Docker 路径相关文档。

## 三点一、配置字段约定
- 在 `cloud_training_config.json` 的 `server` 节点增加 `runtime_mode` 字段。
- 允许值：
  - `preinstalled_pytorch`
  - `docker_ubuntu2204`
- 老配置兼容策略：若缺失或非法值，自动回退为 `preinstalled_pytorch`。

## 四、迁移策略
- 第 1 阶段（已完成）：创建目录骨架与占位文件。
- 第 2 阶段：新增 Docker 资产到 `deploy/docker_ubuntu2204/`，不改主流程行为。
- 第 3 阶段：在 GUI 中增加 `runtime_mode` 分流，按服务器类型选择执行路线。
- 第 4 阶段：将历史文档和工具脚本逐步迁移到对应子目录。

## 五、兼容原则
- 不修改现有用户入口与发布方式。
- 不删除旧文件，先稳定运行再迁移。
- 每次迁移只做一类改动，确保可回滚。
