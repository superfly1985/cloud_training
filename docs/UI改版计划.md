│   训练参数              │  事件进度                                       │

## 二、各区域详细设计

### 2.1 服务器设置（顶部左侧，280px）

**移除项**
- 服务器详情（弹窗按钮）→ 合并入本卡片
- 路径输入（已移到数据集区域）

**保留/新增控件**

| 控件 | 类型 | 说明 |
|------|------|------|
| 连接状态 | Label + 颜色指示 | 红色=未连接 / 绿色=已连接 |
| 连接按钮 | Button | 蓝色，text="连接"/"断开" |
| 主机名 | Label（连接后显示） |  |
| 操作系统 | Label（连接后显示） |  |
| CPU型号 | Label（连接后显示） |  |
| CPU核心数 | Label（连接后显示） |  |
| GPU型号 | Label（连接后显示） | 无GPU时显示"无独立GPU" |
| 显存 | Label（连接后显示） | 无GPU时显示"-" |
| 内存总量 | Label（连接后显示） |  |
| 磁盘总量 | Label（连接后显示） |  |
| Ping延迟 | Label（连接后实时显示） | 单位 ms，1s刷新 |
| 环境检查按钮 | Button（次要按钮） | 灰色描边，text="环境检查" |
| 修复环境按钮 | Button（橙色） | 仅在检查失败后可用，text="修复环境" |

**交互逻辑**
- 连接成功后，并发拉取：主机名/OS、CPU型号、GPU型号、内存/磁盘总量、Ping延迟
- 拉取失败时对应字段显示"获取失败"，不阻塞主流程
- 断开连接后，清空所有主机信息字段

### 2.2 系统监控（顶部中央，flex）

**移除项**
- GPU利用率（峰值/当前）
- GPU显存（已移至服务器设置）

**保留/新增控件**

| 控件 | 说明 |
|------|------|
| GPU利用率实时图表 | 折线图，Y轴 0-100%，每 1s 刷新 |
| CPU利用率实时图表 | 折线图，Y轴 0-100%，每 1s 刷新 |
| 内存使用率实时图表 | 折线图，Y轴 0-100%，每 1s 刷新 |
| 当前数值标签 | 每个图表右下角显示当前数值 |

**图表规格**
- 高度：每个图表 120px
- 宽度：填满容器
- 数据点：保留最近 50 个点
- 配色：GPU=蓝色(#4C78E9)、CPU=橙色(#F59E0B)、内存=绿色(#10B981)
- 背景网格：淡灰色虚线

### 2.3 训练日志（顶部右侧，380px）

**移除项**
- 无

**保留/新增控件**

| 控件 | 说明 |
|------|------|
| 日志文本区 | ScrolledText，高度填满，字体 Consolas 10，绿色前景色 |
| 清除日志按钮 | 次要按钮（小尺寸），text="清空" |

**行为**
- 训练/上传等操作的日志自动追加
- 包含时间戳 `[HH:MM:SS]`
- 最多保留最近 2000 行，超出自动截断头部

### 2.4 训练参数（中部左侧，flex）

**移除项**
- 基础模型（已移到模型管理卡片）
- 学习率优化建议（内联显示改为悬停提示）
- 图像增强（独立分区）

**保留/新增控件**

| 控件 | 类型 | 说明 |
|------|------|------|
| Epochs | Spinbox | 范围 1-10000，默认 300 |
| Batch Size | Spinbox | 范围 1-512，默认 20 |
| Image Size | Spinbox | 范围 320-4096，默认 1024 |
| Learning Rate | Entry | 默认 0.01，支持小数 |
| 快捷推荐 | Button | text="推荐参数"，点击自动填 batch+lr |
| 快速预设 | Combobox | 选项：快速验证(10e/16b/640)/标准训练(300e/20b/1024)/长时训练(1000e/8b/1280) |
| 已有模型 | Combobox | 拉取服务器 /root 下的 .pt 文件，下拉选择 |

**快捷预设明细**

| 预设名 | epochs | batch_size | image_size |
|--------|---------|-------------|------------|
| 快速验证 | 10 | 16 | 640 |
| 标准训练 | 300 | 20 | 1024 |
| 长时训练 | 1000 | 8 | 1280 |

### 2.5 事件进度（中部右侧，flex）

**移除项**

**保留/新增控件**

| 控件 | 类型 | 说明 |
|------|------|------|
| 上传进度条 | Progressbar | 百分比文本覆盖，进度条颜色=蓝色 |
| 上传状态文字 | Label | 显示：未上传 / 上传中 67% (234MB/350MB) / 上传完成 |

- 训练中：开始训练按钮变为"停止训练"（红色），其余 disabled

## 四、按钮配色规范

| 用途 | 颜色 | Bootstrap风格 | 说明 |
|------|------|-------------|------|
| 主操作（连接/开始训练/下载） | 蓝色 | `primary` | 最常用操作 |
| 危险操作（停止/删除/清空） | 红色 | `danger` | 不可逆或高风险 |
| 维护操作（环境修复/TFLite转换） | 橙色 | `warning` | 需要注意但非危险 |
| 次要/检查（环境检查/数据集检查/清空日志） | 灰色描边 | `secondary-outline` | 辅助性操作 |
| 成功状态提示 | 绿色 | `success` | 状态/进度完成 |

## 五、分阶段实施计划

### 第1步：纯布局重排（不改动业务逻辑）
- 调整 `__init__` 中的 `root.geometry("1270x900")`
- 重构 `setup_ui()` 布局网格
- 将控件按新分区重新放置
- 验证窗口尺寸和控件不溢出
- 编译检查无报错

### 第2步：连接即拉取主机信息
- 连接成功后并发执行轻量查询命令
- 将结果显示在服务器设置卡片中
- Ping延迟实时刷新

### 第3步：统一按钮风格和状态联动
- 按配色规范统一所有按钮样式
- 实现完整的按钮 enabled/disabled 联动逻辑

### 第4步：TFLite独立Tab和日志分段
- 确认 TFLite 转换已有独立 Tab
- 优化日志输出格式

### 第5步：收尾和打包验证
- 整体功能回归测试
- 打包测试（PyInstaller onedir）
- 安装到本地设备验证

## 七、注意事项

1. **不引入新依赖**：继续使用 `ttkbootstrap`，不引入 PyQt-Fluent-Widgets
2. **打包体积优先**：每次改动后确认不增加不必要的大依赖
3. **增量交付**：按阶段交付，每步完成后验证再进入下一步
4. **日志可读性**：长日志自动截断或分段，避免刷屏
5. **SSH连接复用**：尽量复用 `self.ssh_client`，避免频繁新建连接
